装着 Brightcove ビデオ Video Cloud Sitecore CMS API

このトピックでは、 Brightcove ビデオ接続モジュール。

インストール Video Cloud Sitecore メディアフレームワーク2.1

Brightcove ビデオ接続には必須 Video Cloud Sitecore メディアフレームワークを最初にインストールします。 ザ Video Cloud Sitecore Media Frameworkは、 Video Cloud Sitecore インストールウィザード。 インストールウィザードで表示される指示に従います。

ワークフローからページ・イベントを解放する

挽き目 Video Cloud Sitecore Media Frameworkには、ワークフローから解放して使用する前に公開する必要のある多数のページイベントが含まれています。

  1. Video Cloud Studioで Video Cloud Sitecore ワークボックス
  2. アナリティクスのワークフローを選択する
  3. 次のページイベントを展開します。
    • 再生が変更されました
    • 再生が完了しました
    • 再生エラー
    • 再生開始
  4. ページイベントを公開する

インストール Brightcove ビデオ Video Cloud Sitecore 経験プラットフォーム

Brightcove Video Connect for Video Cloud Sitecore は、 Video Cloud Sitecore インストールウィザード。 インストールウィザードで表示される指示に従います。 ザ Video Cloud Sitecore プラグインは上記の表のリンクからダウンロードできます。 このzipファイルには、ソースパッケージとプラグインパッケージが含まれています。 最初に解凍してから、内部ZIPパッケージを Video Cloud Sitecore インスタンス。

データベース構成のレポート

ためには Video Cloud Sitecore Media Frameworkレポートが機能するようにするには、エクスペリエンスデータベース（xDB）レポートデータベースに新しいファクトテーブルを追加する必要があります。 この新しいテーブルを作成するスクリプトは、SDNで使用できます。 このスクリプトは、xDBレポートデータベースで実行する必要があります。 この手順は、データベース管理者が完了する必要があります。

注：インストールすることは可能です複数 Video Cloud Sitecore メディアフレームワークコネクタ。 このスクリプトは、インストールしたコネクタの数に関係なく、レポートデータベースごとに1回だけ実行する必要があります。

マルチサーバー構成

Video Cloud Sitecore Media Frameworkは、マルチサーバーコンテンツオーサリング環境にインストールすることができます。 もしあなたの Video Cloud Sitecore 環境がこのように構成されている場合は、以下を構成する必要があります。

コンテンツ管理サーバー

スケジューラを使用してメディアを同期する場合は、スケジューラを構成する必要があるコンテンツ管理（CM）サーバは1つだけです。 スケジュールされたジョブを他のCMサーバの設定ファイルから削除します。 ザ Video Cloud Sitecore イベントキューは、他のCMサーバーを同期させた状態に保ちます。

注：複数のCMサーバでスケジューラを設定しても問題は発生しませんが、そのようなメリットはありません。

コンテンツ配信サーバー

各コンテンツ配信（CD）サーバーで、指定された構成ファイルから次の索引を削除します。 このインデックスは、CM環境でのみ必要です。

ファイル： Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
インデックス名： MediaFramework_Brightcove_Index

CDサーバを設定するときは、CMサーバから設定ファイルをコピーすると便利です。 これを行い、メディアを同期するスケジューラを追加した場合は、スケジュールされたジョブをCDサーバーの設定ファイルから削除する必要があります。 メディアの同期は、CDサーバー上では実行しないでください。 1台のCMサーバで実行する必要があります。 パブリッシュは、メディアアイテムをCDサーバー上で利用できるようにするために使用されます。

定義 Video Cloud Players の作成と管理

単一のビデオまたはビデオの再生リストを表示するには、プレーヤーが必要です。 プレーヤーは、ビデオを見たいビジターがどのような機能を利用できるかを定義します。 Video Cloud Sitecore Media Frameworkを使用すると、コンテンツ作成者は新しいプレーヤーを追加したり、既存のプレーヤーを編集したり、既存のプレーヤーを削除したりすることができます。

プレーヤーを追加する

このセクションでは、 Video Cloud Sitecore プレーヤーに対応するアイテム。

  1. In Brightcove Video Cloud プレーヤーセクションに移動します。
  2. あなたが作成するプレイヤーを選択してください Video Cloud Sitecore 。 プレーヤーIDの値に注意してください。 これを入力する必要があります Video Cloud Sitecore 次のセクションで説明します。
  3. In Video Cloud Sitecore 、メディアライブラリで、 Brightcove アカウント項目。
  4. アカウント項目を選択します。
  5. Players項目を選択します。
  6. 新たに追加 Brightcove ビデオプレーヤーアイテム。 プレーヤーIDの入力を求められます。 からの値を入力 Brightcove Video Cloud .

デフォルトプレーヤーを設定する

動画や再生リストをプレビューするには Video Cloud Sitecore デフォルトプレイヤーを指定する必要があります。 このセクションでは、デフォルトプレーヤーを設定する方法について説明します。

  1. In Video Cloud Sitecore 、メディアライブラリで、 Brightcove アカウント項目。
  2. アカウント項目を選択します。
  3. 設定項目を選択します。
  4. [デフォルトのビデオプレーヤー]フィールドと[デフォルトのプレイリストプレーヤー]フィールドの値を設定します。
  5. アイテムを保存する

プレーヤーを削除

プレーヤーアイテムは他のプレイアイテムと同様に削除されます Video Cloud Sitecore アイテム。

動画のインポート、追加、プレビュー、および削除

Video Cloud Sitecore Media Frameworkを使用すると、コンテンツ作成者はローカルマシンからビデオをアップロードしたり、既存のビデオを編集したり削除したりすることができます。 新しいタグを定義してビデオコンテンツを整理することができます。

ビデオの読み込み

A Video Cloud Sitecore コンテンツ作成者または管理者は、メディアインポートプロセスを手動でトリガーすることができます。

ビデオをインポートすると、 Video Cloud Sitecore 動画のアップロード日ではなく、動画がインポートされた当日を反映します Video Cloud 。 既存のものを移動する場合 Video Cloud Sitecore 以前のコンテンツを移行することができ、コネクタはこれらのビデオを認識します。

挽き目 Brightcove Video Connectプラグインには、定期的にメディアを自動的に同期するエージェントが含まれています。 エージェントは、次の場所に定義されています。

設定ファイル： Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
エージェント名： MediaFramework_Import_Brightcove
  1. トップメニューを右クリックし、Media Frameworkを選択してMedia Frameworkリボンを表示します。
  2. Media Frameworkリボンをクリックします。
  3. [すべてインポート]ボタンをクリックします。
  4. インポートプロセスが実行されると、ステータスボックスが表示されます。 ステータスボックスが消えると、インポートプロセスは完了です。
  5. メディアがインポートされたことを確認するには、[検索]タブをクリックします。
  6. 検索を実行します。
  7. 動画をフィルタリングします。
  8. 検索結果が表示されます。

ビデオを追加します

Video Cloud Sitecore Media Frameworkでは、コンテンツ作成者は新しいビデオを追加できます。

  1. Video Cloud Studioで Video Cloud Sitecore メディアライブラリ。
  2. プレイリストを作成する動画を含むアカウントに移動します。
  3. メディアコンテンツアイテムに移動します。
  4. ナビゲーションヘッダーの Brightcove ビデオ。
  5. [ファイルを追加]をクリックします。
  6. ローカルマシンからアップロードするファイルを選択します。
  7. [スタート]ボタンをクリック。
  8. 動画がアップロードされたら、アップローダウィンドウを閉じます。

タグを定義する

Video Cloud Sitecore Media Frameworkでは、コンテンツ作成者は新しいタグを追加できます。

  1. Video Cloud Studioで Video Cloud Sitecore メディアライブラリ。
  2. タグを作成する動画を含むアカウントに移動します。
  3. タグ項目に移動します。
  4. ナビゲーションヘッダーの Brightcove タグ。
  5. 項目名を入力します。 これがタグになります。

ビデオとタグの編集

動画やタグのアイテムは他のアイテムと同様に編集されます Video Cloud Sitecore アイテム。

ビデオとタグを削除する

動画やタグアイテムは他のアイテムと同様に削除されます Video Cloud Sitecore アイテム。

動画や再生リストは、名前と説明でテキスト文字列を検索することで見つけることができます。 ビデオは、カスタムフィールドでテキスト文字列を検索することによっても見つけることができます。

検索

[検索]タブをクリックして、メディアを検索します。 名前または説明フィールドで検索するテキストを入力します。

虫眼鏡をクリックして検索を実行します。

フィルタリング

検索結果が表示されます。

プレイリストの読み込み、追加、プレビュー、および削除

プレイリストは、単一のプレーヤーのビデオの集まりです。 Video Cloud Sitecore Media Frameworkを使用すると、コンテンツ作成者は新しいプレイリストを追加したり、既存のプレイリストを編集したり、既存のプレイリストを削除したりすることができます。

プレイリストをインポートすると、 Video Cloud Sitecore その中に作成日付ではなく、再生リストがインポートされた現在の日が反映されます Video Cloud 。 既存のものを移動する場合 Video Cloud Sitecore 以前のコンテンツを移行することができ、コネクタはこれらのビデオを認識します。

プレイリストを追加

Video Cloud Sitecore Media Frameworkでは、コンテンツ作成者は新しいプレイリストを追加できます。

  1. Video Cloud Studioで Video Cloud Sitecore メディアライブラリ。
  2. 再生リストを作成する動画を含むアカウントに移動します。
  3. メディアコンテンツアイテムに移動します。
  4. 新たに追加 Brightcove プレイリストアイテム。
  5. [名前]フィールドには、動画のわかりやすい名前を入力します。
  6. [動画ID]フィールドでは、プレイリストに追加する動画を選択します。

プレイリストを編集

プレイリストアイテムは他のプレイリストアイテムと同様に編集されます Video Cloud Sitecore アイテム。

プレイリストを削除

プレイリストアイテムは他のアイテムと同様に削除されます Video Cloud Sitecore アイテム。

ビデオを追加する Video Cloud Sitecore Website

Video Cloud Sitecore Media Frameworkを使用すると、コンテンツ作成者はさまざまな方法でメディアにページを挿入できます。 この章では、使用可能なオプションについて説明します。

リッチテキストエディタ

リッチテキストフィールドを使用すると、コンテンツ作成者は書式設定されたコンテンツをページに挿入できます。 コンテンツ作成者は、リッチテキストエディタを使用して、このフォーマットされたコンテンツを作成および編集します。 このセクションでは、 Video Cloud Sitecore Media Frameworkは、リッチテキストエディタに追加されます。

メディアを埋め込む

メディアはリッチテキストフィールドに直接埋め込むことができます。

  1. リッチテキストエディタで、[メディアを埋め込む]ボタンをクリックします。
  2. 検索画面を使用して、挿入するメディアを探します。 挿入するメディアを表す行をクリックします。 [次へ>]ボタンをクリックします。
  3. 選択したメディアの種類に応じて、追加のオプションが表示されます。 たとえば、ビデオを選択する場合は、ビデオの表示に使用するプレーヤと、プレーヤのサイズを指定する必要があります。 Finishをクリックします。
  4. メディアはリッチテキストフィールドに埋め込まれています。

メディアリンクを埋め込む

リッチテキストフィールドにメディアへのリンクを挿入することができます。

  1. リッチテキストエディタで、[メディアリンクを埋め込む]ボタンをクリックします。
  2. 検索画面を使用して、挿入するメディアを探します。 挿入するメディアを表す行をクリックします。 [次へ>]ボタンをクリックします。
  3. リンクタイトルの値を入力します。 これは、メディアリンクに表示されるテキストです。
    さらに、選択したメディアのタイプに応じて、追加のオプションが表示されます。 たとえば、ビデオを選択する場合は、ビデオの表示に使用するプレーヤと、プレーヤのサイズを指定する必要があります。 Finishをクリックします。
  4. リッチテキストフィールドにリンクが埋め込まれています。

レンダリング

レンダリングにより、ページ上の特定の場所にコンテンツを配置することができます。 Video Cloud Sitecore Media Frameworkには、コンテンツ作成者がメディアをページに挿入できるレンダリングが含まれています。

埋め込みメディアレンダリング

  1. ページエディタを使用して、プレースホルダにレンダリングを追加します。
  2. 埋め込みメディアレンダリングを選択します。
  3. レンダリングがプレースホルダに追加されました。 次に、[メディア要素を埋め込む]ボタンをクリックします。
  4. 検索画面を使用して、挿入するメディアを探します。 挿入するメディアを表す行をクリックします。 [次へ>]ボタンをクリックします。

再生イベント分析

挽き目 Video Cloud Sitecore Media Frameworkを使用すると、ビデオ再生イベントを Video Cloud Sitecore デジタルマーケティングシステム（DMS）。 このセクションでは、使用可能なオプションについて説明します。

再生イベント

再生イベントは、ビデオプレーヤーの操作の結果としてDMSページイベントを記録するために使用されます。 たとえば、ビデオ再生が開始されると、DMSページイベントを書き込むことができます。

デフォルトの再生イベント

Video Cloud Sitecore Media Frameworkには、次の再生イベントが付属しています。

  • 再生開始
  • 再生が完了しました
  • 再生エラー

アカウントの再生イベントを設定する

デフォルトでは、再生開始イベントと再生完了イベントが記録されます。 これは、[設定]項目の Brightcove アカウント項目。

設定項目には、再生イベントルールという名前のフィールドがあります。 動画や再生リストに指定されているイベントを記録するようにMedia Frameworkに指示するルールが定義されています。

アカウントの再生イベントを無効にする

特定のアカウントのすべての再生イベントを無効にすることができます。

  1. コンテンツエディタで、[メディアライブラリ]> [メディアフレームワーク]> [アカウント]の下のアカウントに移動します。
  2. 設定項目を選択します。
  3. Playback Events Rulesフィールドには、再生イベントを登録するために満たす必要がある条件が記述されています。 デフォルトでは、条件は次のとおりです。 true条件を変更するか、既存の規則を削除するには、[規則の編集]ボタンをクリックします。

メディアの再生イベントを指定する

各プレイリストとビデオアイテムにはEventsというフィールドがあります。 このフィールドは、指定された項目に対してどの再生イベントが処理されるかを決定します。

次のスクリーンショットは、ビデオの再生イベントが指定されていることを示しています。

注：標準値は、すべてのプレイリストとビデオに対して特定のイベントが記録されるようにするために使用できますが、このフィールドでは個々のプレイリストやビデオに独自のイベントを選択させることもできます。

注：これらのイベントは、[設定]項目が Brightcove アカウントアイテムは、「メディアアイテムイベントの追加」アクションで構成された再生イベントルールフィールドを有する。 セクション3.1.2を参照してください。

新しい再生イベントを作成する

新しい再生イベントを追加して、ビデオ再生に関するより詳細な情報を取得することができます。

  1. Content Editorで、[システム]> [モジュール]> [メディアフレームワーク]> [再生イベント]に移動します。
  2. テンプレートの再生イベントを使用して新しいアイテムを追加する
  3. [ページイベント]フィールドで[再生が変更されました]を選択します。
  4. 「パラメータ」フィールドには、次のいずれかの形式で値を入力できます。
    1. Number - ビデオがこの秒数で再生された後、ページイベントが記録されます。 有効な値の例は15です
    2. パーセント - ビデオがこの量で再生された後、ページイベントが記録されます。 有効な値の例は、50％
  5. アイテムを保存する
  6. アイテムを公開する
  7. これで新しいページイベントをプレイリストとビデオに追加することができます

注：プレイリストとビデオに新しい再生イベントを割り当てることを忘れないでください。 それ以外の場合、イベントは記録されません。
