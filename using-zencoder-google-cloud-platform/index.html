Google Cloud StorageはAmazon S3 APIと相互運用可能ですが、初期設定作業が必要です。 GCSを使い始めるには Zencoder、あなたは有効にする必要があります 相互運用アクセス Google Cloud Consoleから送信し、鍵を生成します。 チェックアウト 開発者キーを生成するには のセクション この文書 相互運用可能なアクセスをオンにしてアクセスキーを生成する方法については、

ストアの資格情報

Once アカウントの相互運用可能なGCS資格情報のセットを持っている場合、それらを安全に保存できます。 Zencoder、 Zencoder 証明 のページをご覧ください。

下 資格の追加選択 Googleのクラウドストレージ。 次に、 Google Cloud Storage Access Key ID や Secret Access Key パラメータを新しく生成された資格情報で置き換えます。 「gcsを使用した転送のデフォルトの認証情報」を必ずクリックしてください。これにより、 gcs:// プロトコルはこれらの資格情報を使用します。

トランスコード

GCSクレデンシャルのセットとトランスコードするために保存されたコンテンツを備えた次のステップは、GCE（Google Computing Engine）でジョブの実行を開始することです。 設定することにより 地域 パラメータを使用すると、ジョブごとに実行されるクラウド領域のトランスコードジョブを制御できます。 GCEの新しい地域名は "us-central-gce"と "eu-west-gce"です。

以下の例は、GCSから入力ビデオを取り出し、HTTP用のレンディションを作成する作業ジョブを示しています Live ストリーミングし、出力ストリームをGCSに送信します。 あなたはこれを変更することでテストすることができます my-bucket あなたが所有している既存のGCSバケットに リクエストビルダー. すべての既存の Zencoder パラメータとストレージプロバイダは、Rackspace Cloud FilesやAmazon S3などの新しいGCE領域で動作します。
