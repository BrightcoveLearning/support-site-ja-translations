このトピックは、Embed APIをいつどのように使用するかを決定するのに役立ちます。 プレーヤー設定APIと埋め込みAPIのどちらを使用するかの選択は重要な決定であり、このドキュメントの内容がこれらの決定をガイドします。

このガイドのカールに関する記述は、 クイックスタート：プレーヤー管理。 プレーヤー管理のクイックスタートを実行した後でこのガイドを操作すると、子プレーヤーに加えられた変更と、重要なことに親プレーヤーから継承された機能を確認できます。

埋め込みAPIを使用する理由

埋め込みAPIを使用すると、特定のプレーヤーの複数のインスタンスを作成できます。 このプレーヤー/インスタンスの関係を親/子の関係と考えるのがよいでしょう。 シングルプレーヤーが親であり、埋め込みAPIで作成されたプレーヤーは親プレーヤーの子です。 親プレーヤーには、プレーヤーに必要なプロパティの大部分が含まれているため、embed APIを使用して、さまざまな子プレーヤーのプロパティのサブセットをカスタマイズできます。 たとえば、さまざまなメディアをロードしたり、さまざまなプラグインを使用したり、さまざまな子プレーヤーでスタイルを設定したりできます。

次の図は、機能を明確にするのに役立ちます。 親の下には左が表示され、2人の子プレーヤーが右に表示されます。 次のことを確認してください。

  • ポスターは両方の子供に受け継がれます
  • 再生ボタンの形状は上の子によって継承され、下の子ではオーバーライドされます
  • 一番上の子はプロパティを追加します。この場合、オーバーレイは親が持っていません

この親子関係のもう1つの強力な機能は、継承が進行中であることです。 次の図は、親に割り当てられた新しいポスターを示していて、両方の子がその構成の変更を継承します。

注：デフォルトでは、親プレーヤーごとに50人の子プレーヤーに制限されています。 さらに多くのユースケースがある場合は、サポートに連絡し、状況について話し合ってください。

組み込みAPIを使用しない場合

ユースケースで必要な場合に埋め込みAPIを使用する理由はいくつかありますが、通常のプレーヤーに固執する理由もいくつかあります。 ここにいくつかあります：

  • 子どものプレイヤーは、編集することはできません Video Cloud スタジオ。 子プレーヤーを編集できるのは、 Player Management API。 あなたは子供のプレーヤーの親プレーヤーを編集することができます Video Cloud しかし、スタジオでは、親プレーヤーに加えられた変更がすべての子プレーヤーに影響します。
  • 親プレーヤーに関連付けられている子プレーヤーが多い場合、親プレーヤーの公開には時間がかかることがあります。 各子プレーヤーは個別に公開されます。30を超える子プレーヤーがある場合は、子プレーヤーの公開に多少の遅延が予想されます。 これは、30人のレギュラープレーヤーを同時に公開する場合とまったく同じです。

上記の理由を考えると、通常のプレーヤーを使用することから始めて、子プレーヤーの必要性がわかったときに埋め込みを試すことが理にかなっています。

ビデオタグデータ埋め込み

親子プレイヤーには表記上の違いがあります。 標準のページはめ込みプレーヤーコードがこの形式で表示されます：

<video-js
  data-account="1507807800001"
  data-player="HiAdwRZ7kK"
  data-embed="default"
  controls=""
  data-application-id=""
  class="vjs-fluid"></video-js>

挽き目 data-embed 属性は、プレーヤが親であるか子であるかを決定する。 値が defaultプレーヤーは親です。 プレーヤーが子供の場合、 data-embed 属性には、親プレーヤーのIDが含まれます。 その例を次に示します。

挽き目 data-embed 属性は、プレーヤが親であるか子であるかを決定する。 値が defaultプレーヤーは親です。 プレーヤーが子供の場合、 data-embed 属性には、親プレーヤーのIDが含まれます。 その例を次に示します。

<video-js
  data-account="1507807800001"
  data-player="HiAdwRZ7kK"
  data-embed="NURK56ZSV"
  data-application-id=""
  class="video-js" controls></video-js>

なお、 data-player、つまりプレーヤーIDは同じですが、 data-embed から変更されました default 子プレーヤーのID。

子プレーヤーのURL

親プレーヤーと子プレーヤーをどのように区別しますか？ URLは異なります。 たとえば、親プレーヤーのURLは次のとおりです。

//players.brightcove.net/1507807800001/HiAdwRZ7kK_default/index.min.js

埋め込みAPIを使用して子プレーヤーを作成した後、次に示すように、子プレーヤーのIDが親のURLに追加されます。

//players.brightcove.net/1507807800001/HiAdwRZ7kK_NURK56ZSV/index.min.js

親/子ユースケース

複数のビデオプレーヤーを使用すると仮定します。 多くの場合、プレーヤーの共通機能はほとんど同じですが、特別な場合にはプレーヤーを調整する必要があります。 Player Configuration APIを使用して複数のプレーヤーを作成することができます POST や PATCH メソッド、しかしこれは重大なメンテナンス問題につながる可能性があります。 たとえば、すべてのプレイヤーのポスターを変更したいとします。 これは使用することを意味します PATCH すべての異なるプレーヤーに 一方、子プレーヤーを作成した場合は、 PATCH 親プレーヤーとすべての子プレーヤーが自動的に新しいポスターを持っています。

作成プロセス

あなたが ステップバイステップ：プレイヤー管理 curlステートメントを使用してHTTPメソッドを Player Management API。 ここでも同じアプローチを使用します。

プレーヤを作成するには、プレーヤコンフィグレーションAPIでいくつかのHTTPメソッドを使用している可能性があります。

  • プレーヤーを作成する POST 〜へ https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players
  • プレーヤーを更新する PATCH 〜へ https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration
  • 更新されたプレーヤーをaを使用して公開する POST 〜へ https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/publish

埋め込みAPIを使用する子プレーヤーにも同様のアプローチが使用されます。 非常に高いレベルであなたは：

  • 子プレーヤーを作成するには、 POST 〜へ https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/埋め込む。 注：埋め込みAPIを使用して作成されたチャイルドプレイヤーは作成時に自己公開するため、子プレーヤーの作成時には公開する必要はなく、子プレーヤーの更新時にのみ公開します。
  • 子プレーヤーを更新する PATCH 〜へ https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/embeds / $ EMBED_ID / configuration
  • 子プレーヤーをaを使用して公開する POST 〜へ https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/embeds / $ EMBED_ID / publish

以下の内容はプロセスを詳細に説明しています。

子プレーヤーを作成する

子プレーヤーを作成するには、HTTPを使用します POST メソッドを呼び出します。

curl /
--header "Content-Type: application/json" /
--user $EMAIL /
--request POST /
--data '{
"media": {
"sources": [
  {
    "src":"http://solutions.brightcove.com/bcls/assets/videos/BirdsOfAFeather.mp4",
    "type":"video/mp4"
  }
],
"poster": {
  "highres":"http://solutions.brightcove.com/bcls/assets/images/BirdsOfAFeather.jpg"
}
}
  }' /
https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/embeds

子プレーヤーの作成に対する応答の例は次のとおりです。

    "embed_in_page": "http://players.brightcove.net/1507807800001/668c5107-a80c-4940-8c17-279c01ce101c_be864624-8d85-4dfc-8fe6-4e9dd4c70417/in_page.embed",

注：子プレイヤーは作成時に自己公開しますので、作成後に子プレイヤーを公開する必要はありません。 子どものプレーヤーが変更された場合でも、それを公開する必要があります PATCH 方法。 この時点で、プレビュー情報は、作成された直後に公開された子プレーヤーを使用できるので役立ちません。

これで、 url 結果を見るために子供のプレーヤーからプロパティ。 以下の例では、ステップバイステップの「プレイヤー管理」で作成した親プレイヤーに子プレイヤーが追加されました。 あなたは新しいポスターとビデオを見ます、しかし親プレーヤーからのオーバーレイプラグインはまだ存在しています。

子プレーヤーを更新する

子プレーヤーを更新するには、HTTPを使用します PATCH 方法。 次のcurlステートメントは、 poster プロパティ。 あなたが $EMBED_ID 環境変数を適切に：

curl
--header "Content-Type: application/json"
--user $EMAIL
--request PATCH
--data '{
"media": {
"poster": {
  "highres":"http://solutions.brightcove.com/bcls/assets/images/Water-Splashing.jpg"
}
}
  }'
https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/embeds/$EMBED_ID/configuration

レスポンスは、 preview_url や preview_embed_code コー​​ド：

{
    "preview_url": "http://preview-players.brightcove.net/1507807800001/668c5107-a80c-4940-8c17-279c01ce101c/be864624-8d85-4dfc-8fe6-4e9dd4c70417/index.html",
    "preview_embed_code": "<iframe src='//preview-players.brightcove.net/1507807800001/668c5107-a80c-4940-8c17-279c01ce101c/be864624-8d85-4dfc-8fe6-4e9dd4c70417/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>"
}

子プレーヤーを公開する

Once 子プレイヤーが変更された場合は、公開する必要があります。 必ず $EMBED_ID 環境変数が設定され、新しく変更された子プレーヤーを公開できます。

curl
--header "Content-Type: application/json"
--user $EMAIL
--request POST
https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/embeds/$EMBED_ID/publish

このレスポンスは、プレイヤーの公開に非常によく似て、子プレーヤーを使用するために必要な重要な情報を提供します。

{
    "id": "be864624-8d85-4dfc-8fe6-4e9dd4c70417",
    "url": "http://players.brightcove.net/1507807800001/668c5107-a80c-4940-8c17-279c01ce101c_be864624-8d85-4dfc-8fe6-4e9dd4c70417/index.html",
    "embed_code": "<iframe src='//players.brightcove.net/1507807800001/668c5107-a80c-4940-8c17-279c01ce101c_be864624-8d85-4dfc-8fe6-4e9dd4c70417/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "embed_in_page": "http://players.brightcove.net/1507807800001/668c5107-a80c-4940-8c17-279c01ce101c_be864624-8d85-4dfc-8fe6-4e9dd4c70417/in_page.embed"
}

子情報を表示する

HTTPを使用することができます GET メソッドを使用して、子プレーヤーに関する情報を取得します。 curlステートメントの例は次のとおりです。

curl
  --header "Content-Type: application/json"
  --user $EMAIL
  --request GET
  https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/embeds

非常に大量のJSONデータが返されます。

子プレーヤーを削除する

子プレーヤーを削除するには、 DELETE 方法。 子プレーヤーを削除するcurlステートメントの例を次に示します。

curl
--header "Content-Type: application/json"
--user $EMAIL
--request DELETE
https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/embeds/$EMBED_ID

もちろん、これは子プレイヤーだけに影響し、親プレイヤーには影響しません。
