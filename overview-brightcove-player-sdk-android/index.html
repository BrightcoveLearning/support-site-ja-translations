サポートレベルの理解

サポートは、 [メジャーバージョン].[マイナーバージョン] （例えば、Android 8.1など） Googleの最新の技術向上を利用するため、Googleでは次の2つのレベルのサポートを提供しています。

  • 現在行われているキャンペーン: Brightcove これらのOSバージョンでバグをテストして修正します。
  • パッシブ: Brightcove これらのOSバージョンでテストしますが、バグ修正はケースバイケースで処理されます。

一般的には、 Brightcove ユーザーベースの80％で使用されているOSのバージョン、およびユーザーベースの少なくとも5％で使用されている他のバージョンのパッシブサポートをアクティブにサポートしています。

はじめに

Android SDKを使用して開発を始める前に、いくつかの概念を学びます。

  1. 언론방송

    Native Player SDKの現在のバージョンでは、 Video Cloud アカウントを通じて Playback API、インターネットアクセス可能なURLを持つリモートアセットとして使用できます。

  2. アナリティクス

    Native Player SDKを使用すると、 Video Cloud 顧客は自動的に堅牢な分析データを取得します Video Cloud.

  3. 受け台

    GradleはAndroid SDKビルドシステムの基礎であり、Android Studioと統合されています。

  4. メイベンレポ

    挽き目 Brightcove Android Mavenアーティファクト（jarファイル）のネイティブプレーヤーは、 Brightcove Maven Repo - https://repo.brightcove.com/releases

    AndroidプロジェクトにMavenレポを追加する方法の詳細については、 Android用Native SDKを使用してアプリを作成する の資料をご参照ください。
  5. Android SDKの概要

    SDKの理解を深めるために、ネイティブのビデオプレーヤーを制御する複雑さに対処するために重要なコンポーネントがどのようにやり取りされるかを学びます。

  6. Android用Native SDKを使用してアプリを作成する

    あなたの最初のアプリを作成する Brightcove Player 独自のバックエンドシステムからビデオを再生するAndroid用SDK。 Video Cloud 顧客はさらに、 Brightcove カタログサービス。

  7. SDK開発者フォーラム

    SDK関連の質問に回答するSDK開発者フォーラムに参加してください Brightcove エンジニアとコミュニティ。 以前に回答した質問を検索するか、新しい質問を投稿します。 パブリックグループとして、電子メールアドレスを入力することでアクセスできます。

  8. ビデオオンデマンドトレーニング

    一連のトレーニング動画を見ると、Android Studioでアプリを制作し、あなたの動画や再生リストを再生する手順を順を追って説明します。 Video Cloud としょうかん。

ノート

次の注意事項は、ネイティブSDKの最新バージョンに関するものです。

  • バージョン6.10.0以降、 BrightcoveAndroidのNative SDKは、元のサポートライブラリの代わりにJetpack（androidx。*）ライブラリを使用します。 この変更は、Exoplayerバージョン2.10.0以降で必要です。 Jetpackを使用するには、アプリが次の要件を満たしている必要があります。

    • Jetpack（AndroidX）の依存関係に移行する
    • Android Gradle Tools 3.2.0
    • Gradleバージョン4.6

    JetPackへの移行方法の詳細については、公式サイトを参照してください Android移行ガイド.

  • によってサポートされているAndroidの最小SDKバージョン Brightcove Android用ネイティブSDKは19になりました。
  • ExoPlayer v2.9.0 +の場合、Java 8サポートを有効にする必要があります。 これを行うには、以下を追加します android アプリのセクション build.gradle ファイル。 詳細については、 AndroidのExoPlayer の資料をご参照ください。 
    android {
    compileOptions {
      targetCompatibility JavaVersion.VERSION_1_8
    }
  }

    これにより、プロジェクトのデフォルトのJavaバージョンが上書きされます。 compileSdkVersion 値。

ExoPlayer 1のサポートは、AndroidのネイティブSDKから削除されました。 まだ行っていない場合は、に移行してください ExoPlayer 2フレームワーク.

コアSDKのインストール

のインストール Brightcove コアSDKは非常に簡単です。 これを行うには2つの方法があります：

  • gradleビルドシステムの使用
  • 手動でSDKをダウンロードする

gradleビルドシステムの使用

追加する最も簡単な方法は Brightcove SDKとその依存関係は、Android Studioでgradleビルドシステムを使用することです。

Gradleコミュニティでは、「+」文字を使用した動的依存関係バージョンの使用は推奨されていません。 動的バージョン管理を使用すると、新しいAPIがアプリのソースコードと予期せず互換性がなくなる可能性があるため、ビルドプロセスに重大なリスクをもたらします。

Brightcove 特定のバージョン文字列で常に最新バージョンを使用することをお勧めします。 たとえば、 build.gradle や gradle.properties のファイル アンドロイドプレイヤーサンプル プロジェクト。

gradleの使用方法の詳細については、Androidの ビルドの依存関係を追加する ドキュメンテーション。

手動でSDKをダウンロードする

手動でダウンロードすることもできます Brightcove SDKは次のようになります。

  1. SDKをダウンロードして解凍するだけです。 いくつかのフォルダがあります。
  2. 内部 SDDK フォルダには、名前を持つコアSDKがあります android-sdk-xxxaarここで、 xxx 最新のSDKバージョン番号です。
  3. 加えます android-sdk-xxxaar あなたのライブラリに必要なすべてのプラグインファイルがあります。

コンパニオン広告

コンパニオン広告とクリックスルー広告は、Android TVではサポートされていません。 Android TVではWebブラウザーのサポートがないため、リンク先URLのリンク先はありません。

ProGuardを使って作業する

使い方 ProGuard より安全なアプリケーションを作成することができ、リバースエンジニアリングが難しくなります。 ProGuardはAndroidビルドシステムに統合されているため、ProGuardを有効にするか、ビルドせずにビルドするかを選択できます。

ProGuardがGradleで有効になっているときにエラーが発生した場合は、次の除外ルールを追加してみてください。

  -dontwarn com.google.**
  -dontwarn android.media.**
  -keep class android.media.** { *; }
  -keep class com.google.** { *; }
  -keep interface com.google.** { *; }
  -keep class com.google.ads.interactivemedia.** { *; }
  -keep interface com.google.ads.interactivemedia.** { *; }

ProGuardのルールの完全なリストについては、 Brightcove ネイティブSDK、ダウンロードして解凍する 現在のAndroid用Native SDK。 オープン SDDK フォルダを開き、 proguard.txt ファイル。
