このトピックでは、Analyticsモジュールの一部として提供されるパフォーマンスレポートについて説明します。

アナリティクスデータは自動的に Brightcove PlayerNative Player SDKによって提供されるものも含めて、 あなたがいる場合 Studio上ではサポートされていません。 使用して Brightcove Player 配達する Video Cloud データをデータコレクタに送信するために使用しているプレーヤーを計測する必要があります。 を参照してください 概要： Data Collection API 詳細はこちら

掲載結果レポートを表示するには、Analyticsモジュールを開き、 洗浄力 左側のナビゲーションから。 このレポートには、選択した期間の1日の合計ビデオビューを示すグラフが表示されます。 ドロップダウンリストを使用すると、別の指標を選択できます。

  • 動画視聴回数 - ビデオが再生を開始した回数。ストリームの開始時に記録されます（巻き戻しや再生は含まれません）。 個々の視聴者を測定するものではありません。 プレロール広告のある動画の場合、動画の再生が始まるまでビューは記録されません
  • 動画の表示回数 - 動画が読み込まれてインタラクションの準備ができた回数
  • プレイレート - 実際に再生された読み込まれた動画の割合（動画再生回数/動画再生回数）
  • 見た分 - 動画が視聴された時間（分）
  • エンゲージメントスコア - 1回の動画再生あたりの平均視聴率（視聴率を動画再生回数で割ったものを合計したもの）
  • 毎日ユニーク視聴者 - 視聴者のIPアドレスとユーザーエージェントを使用して計算されます。 Brightcove 3rdパーティーのCookieを使用しないことを決定しました。これは、Cookieがブラウザによってブロックされることが増えているためです。 Daily Unique Viewersはアカウント単位でのみご利用いただけます。 このメトリックは動画（またはその他のもの）によるフィルタリングでは使用できません。 Brightcove 独自のIPアドレスではなく、提供する価値に独自の視聴者を置くこともできます。 詳細については、 Analyticsのイベントトラッキング をご参照ください。
  • ビデオ帯域幅 - ビデオの帯域幅のみを含みます。 帯域幅のデータは3-5日遅れることに注意してください。
  • 総帯域幅 - ビデオ、画像、プレーヤーの負荷のための帯域幅を含みます。 帯域幅のデータは3-5日遅れることに注意してください。
ノート：
  • アナリティクスレポートには、アカウントから削除された動画の分析機能が含まれています。 アナリティクスモジュールで削除した動画を検索して、削除した特定の動画に関するレポートを表示することはできません。 このようなレポートを取得するには、削除した動画の動画IDを知っている必要があります。 Analytics API レポートを生成します。
  • デイリーユニークビューアとビデオ帯域幅データのデータが利用可能になるまでに3日かかる。
  • 動画の視聴回数は、エンゲージメントレポートの視聴回数と同じではありません。 ビデオビューは、ビデオが最初に再生を開始するとカウントされます。 1％のビューは、最初のパーセントが監視されているときに記録されます。 違いは、最初のパーセンテージが複数回視聴された場合（たとえば、動画が自動再生に設定されている場合）、カウントが増加するのに対し、動画の再生は最初に再生が開始された時点でカウントされます（ユーザーがページをリロードしたり、プレイリスト内の別の動画を見たりしない限り）リプレイ。
  • 同じアカウント内の同時レポート要求は、連続して実行されます。 レポートの日付範囲が大きく、多くのフィールドが選択されている場合は、すべての結果が返されるまでに時間がかかることがあります。 良い方法は、大きなレポートをスケジュールすることです。
  • ライブストリームイベント開始時間 In-Pageエクスペリエンス、動画がプレーヤーに読み込まれているように見えても、視聴者が再生ボタンオーバーレイをクリックするまで動画は読み込まれません。 したがって、視聴者が動画を再生するまで、動画インプレッションはAnalyticsモジュールによって記録されません。

チャートの下にあるレポートスイッチャを使用して、別のレポートにすばやく切り替えることができます。

レポートスイッチャーの右側にあるドロップダウンリストを使用して、レポートディメンションを選択できます。 パフォーマンスレポートでは、次のディメンションを使用できます。

使用可能なディメンションは、選択されたレポートに基づいて変更されることに注意してください。 レポートの下部には、表示する行数を選択したり、他のページに移動するためのコントロールがあります。

前の期間と比較

選択する 前の期間と比較 チェックボックスは、前の期間のデータを表すオーバーレイ点線グラフを追加します。

前期間は、現在の報告期間に選択されている期間に基づいています。 たとえば、最後の7日の日付範囲を選択すると、前の期間は2週前となります。 週ごとの傾向を比較する場合は、完全な週をカバーする日付範囲を選択することをお勧めします。 たとえば、月曜日から開始する日付を選択する場合は、終了日を日曜日に設定します。 開始日が金曜日の場合は、終了日を木曜日に設定します。 また、棒と破線の両方にマウスを置くと、日付と対応する値が表示されます。

パフォーマンスレポートの詳細

レポートの本文に表示される最初の列は、選択されたレポートのディメンション（動画名、プレーヤー名、国、端末の種類など）に応じて異なります。 すべてのレポートには、次のデータが表示されます。

  • 動画視聴回数 -ビデオが再生を開始した回数。ストリームの開始時に記録されます（巻き戻しや再生は含まれません）。 個々の視聴者の測定ではありません
  • 動画の表示回数 - 動画がプレーヤーに読み込まれ、インタラクションの準備ができた回数
  • プレイレート - 読み込まれた後に動画を再生したユーザーの割合（動画の再生回数を動画の再生回数で割った値）
  • 見た分 - 動画が視聴された総時間数
  • エンゲージメントスコア - 動画の視聴あたりの平均視聴率（すべての視聴率を動画の視聴回数で割った値を加算して計算）

トラフィックソース

トラフィックのソースごとにパフォーマンスを表示すると、可能なトラフィックソースは次のようになります。

  • 紹介 - ユーザーがビデオへのリンクをたどった
  • 直接 - ユーザーがリンクを直接入力したか、参照元を特定できませんでした
  • 安全な検索 - Googleアカウントでログインし、結果を検索してクリックしたユーザー（有料またはオーガニック検索情報の場合はこの場合は含まれません）
  • オーガニック検索 - ユーザーが検索を実行し、結果をクリックして動画にアクセスする
  • 有料検索 - ユーザーが検索を実行し、有料の結果をクリックして動画にアクセスする

レポートデータのエクスポート

レポートデータは、コンマ区切り値とExcel形式の両方でダウンロードできます。 エクスポート機能を使用すると、データを直接 Analytics API コール。 エクスポートされたデータは、UIデータのやり方では操作されず、UIに表示されるフィールドのサブセットだけでなく、レポートの基礎となるすべてのデータが現在含まれています。 UIに表示されるものとエクスポートされるデータの違いのいくつか：

  • 異なる単位（分と秒）
  • 数字の丸め
  • データのエクスポートには、UIのどこに関係なく、パフォーマンスとエンゲージメントの両方の指標が含まれます
  • 挽き目 その他/サードパーティ エントリは関連するビデオIDまたはビデオ名なしでエクスポートされます

すべてのデータをAPIの元の形式にすると、UIで提供されている以外の方法でデータを操作できます。

レポートは印刷用のスタイルシートをサポートしているため、レポートを印刷したりPDFに保存すると、プリンタに適したページが生成されます。
