このトピックでは、ライブストリームからVOD（ビデオオンデマンド）クリップを作成する方法を説明します。

概要

クリップはライブストリームから抽出されたビデオです。 それらは、S3バケット、FTPサイト、または Video Cloud アカウント。 クリップはMP4ビデオとして作成され、すべての場合に宛先に送信されます。 の場合 Video CloudMP4はインジェストシステムによってトランスコードされ、ビデオ用に作成されるレンディションの種類は、使用されるインジェストプロファイルによって異なります。

クリップの設定は、 /vods エンドポイントを使用して定義します

クリップはいくつかの方法で作成できます：

  • ライブストリームイベント開始時間 stream_start_timecode および stream_end_timecode ライブストリームイベントのSMPTEタイムコードで定義されています。 エンコーダがタイムコード情報を送信していることが必要であることに注意してください
  • ライブストリームイベント開始時間 start_time および end_time で定義します（ stream_start_time）より相対的に
  • ライブストリームイベント開始時間 start_time および end_time を Epoch（Unix）時間で定義します。
  • ライブストリームイベント開始時間 duration
  • VOD API 削除されたデジタルマスターを復元することはできません JOB では使用できません 。

ノート

  1. レガシーインジェストプロファイルはライブクリッピングをサポートしなくなりました。 ライブストリームからビデオクリップを作成するには、アカウントでDynamic Deliveryが有効になっていることを確認し、Dynamic Delivery取り込みプロファイルを使用します。
  2. クリップをできるだけ早く利用可能にするために、セグメントに正確なクリップが最初に作成され、利用可能になり次第フレームに正確なクリップに置き換えられます。
  3. 指定した場合 duration結果のクリップは次のようになります。
    • ジョブがアクティブでまだライブの場合：（要求時間 - 持続時間）から（要求時間）
    • ジョブが終了した場合：（ finished_at - 期間）から（ finished_at)
  4. 両方を指定した場合 start_time そして end_time:
    • ジョブがアクティブでまだライブの場合：エポックタイムウィンドウが完全に created_at 要求時間とクリップが作成されます
    • ジョブが終了した場合：エポックタイムウィンドウが完全に created_at や finished_at、クリップが作成されます
  5. 使用しているライブストリームのクリップ SSAI 広告は含まれません。
  6. クリップは、イベントの最大7日後に作成できます。 ために SEP、次のアクティベーション日または7日（いずれか短い方）まで作成できます。
  7. VOD APIは、ストリーム内に存在するコンテンツの外にコンテンツを追加しません。 350秒の長いライブストリームで300を指定すると、出力は300秒になります。
  8. ライブストリームはブロードキャストとして保存され、すぐに、イベント終了後7日間利用可能になるため、クリッピングのためにDVR対応のライブストリームを使用する必要はありません。
  9. Brightcove Live クリッピングは、最高解像度の出力と同じ解像度のクリップのみを生成します。 ソース入力解像度と一致しません（ただし、最高解像度の出力と同じでない場合）。

クリップを複数の宛先に送信することもできます。

  • A Video Cloud アカウント
  • FTPサーバー
  • S3バケット

クリップを指定すると、出力は しなければなりません 含む どちら a url デスティネーション or a videocloud ビデオの作成とクリップの取り込みの詳細を示すオブジェクト Video Cloud.

注：クリップ ライブストリームの実行中に作成することができます。 これを行うには、クリップの開始時刻と終了時刻をエポック時刻または相対時刻で定義する必要があります start ライブストリームの時間。

証明

クリップを送信する宛先にアクセスするための資格情報が必要な場合は、その資格情報操作を使用してこれらを作成できます。 Live API。 見る そのための資格情報の管理 Live API 詳細はこちら

例1：ストリーム開始からの開始/終了時刻

要求本文には、開始時刻と終了時刻、およびクリップの送信先に関する詳細が含まれます。 ここでは、ストリームの3分目のクリップを作成してストリームに送信するサンプルリクエストボディを示します Video Cloud アカウント：

{
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
  "outputs":[
    {
      "label": "60 secs by stream from min 2 to min 3",
      "stream_start_time": 120,
      "stream_end_time": 180,
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
      "videocloud": {
        "video": {
        "name": "One Minute Clip",
        "tags": ["live", "clip"]
        },
          "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

この例では、1分の長さのクリップを作成し、 Video Cloud 。 クリップに名前と2つのタグを付けています。 ingest profile 再トランスコーディングのために、アカウントのデフォルトが使用され、 Video Cloud トランスコード中にクリップからサムネイル画像とポスター画像をキャプチャします。

この例では、特定のEpoch時間（この場合は22 Jan 2018 at 08：24：54 GMT）で1分間のクリップを作成しています。

リクエストボディ - S3

要求本文には、開始時刻と終了時刻、およびクリップの送信先に関する詳細が含まれます。 ストリームの3分目のクリップを作成し、それをS3バケットに送信するサンプルリクエストボディを次に示します。

{
  "live_job_id":"{{job_id}}",
  "outputs":[
    {
      "label": "last_30",
      "duration": 30,
      "url": "s3://YOUR_BUCKET_NAME/file_name.mp4",
      "credentials": "s3-credentials",
      "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
    }
  ],
}

この例では、30秒の継続時間のクリップを作成し、それをS3バケットに送ります。 バケットURLには、クリップのファイル名、保存されたS3バケツクレデンシャルの名前を含む文字列が含まれています。クレデンシャルはアカウントごとに設定できます。 Brightcove サポート。
