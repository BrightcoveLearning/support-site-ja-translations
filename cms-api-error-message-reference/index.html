409 CONCURRENT_UPDATE Update failed because of another update at the same time. Please try again. 別のAPIリクエスト、Studioユーザー、または何らかのシステムプロセスが現在ビデオを更新しています。 これは、同じビデオに対してAPIの更新要求を非同期ですばやく続けて行う場合にも発生する可能性があります。
409 CONFLICT Reference id is already in use. 参照IDの一意性を保証するために、参照IDを含む操作は最大3分の間IDをロックします。 つまり、参照IDを作成または変更する要求を行い、その後すぐにその参照IDを使用する他の要求を行うと、操作は409エラーで失敗します。 1）動画を削除してから、その参照IDを別の動画に割り当てようとしました。 2）新しいビデオを作成しようとして503（service unavailable）エラーで失敗し、同じ要求をもう一度試みます。

操作を再試行する前に、このエラーが表示されてから少なくとも3分待ってください。 また、参照IDを使用して動画を作成しようとしても失敗した場合（参照IDとは無関係の理由で）、その参照IDは30秒間ロックされ、繰り返し作成しようとするまで失敗します。参照IDがロック解除されるまで十分な時間待ちます。
409 CONFLICT Both accounts must be enabled for media sharing. マスターアカウントがメディア共有のためにチャンネルにアフィリエイトアカウントを追加しようとすると、 どちらも メディア共有のアカウントを有効にする必要があります。
409 UNSUPPORTED_MEDIA_TYPE リクエストはヘッダーが欠けている可能性が最も高い： Content-Type: application/json.
415 SHARING_DISABLED Master account is not enabled for media sharing. メディア共有を有効にしていないアカウントからビデオを共有しようとすると、このエラーが返されます。
422 ILLEGAL_FIELD Unrecognized field in the submitted data スペルミスまたは存在しないフィールドの他の使用
422 VALIDATION_ERROR （JSONデータは有効ではありません - エラーメッセージは問題によって異なります） メッセージの例：
  • name：REQUIRED_FIELD（名前付きのビデオを作成し、名前を空の文字列に更新）
  • account_id：WRONG_ACCOUNT（URLのアカウントIDがJSONのアカウントIDと一致しません）
  • 無効なキー：id（IDは変更できません。また、JSONではキーとして表示しないでください）
429 TOO_MANY_REQUESTS Too many requests 同時リクエスト数が多すぎるか、1秒あたりにリクエスト数が多すぎます
500 UNKNOWN an unknown internal error occurred 問題 Brightcove システム - 後でやり直してください。
503 SERVICE_UNAVAILABLE The API is temporarily unavailable バックエンドの問題 - 後でやり直してください。
504 TIMEOUT something took too long サーバーがビジー状態になっている可能性があります。

メディア共有エラー

メディア共有エラーは、APIリクエストに対する個別のエラーレスポンスとして返されるのではなく、 error_message 通常の応答のフィールド：

[
{
"video_id" : "394872349182374",
"affiliate_id" : "234987239487",
"affiliate_video_id" : "30308254055202",
"status" : "COMPLETE",
"shared_at" : "2017-12-11T17:57:45.530Z",
"updated_at" : "2017-12-11T18:03:32.789Z",
"error_message" : "[{"error_code":"MISSING_CUSTOM_FIELDS","error_message":"Affiliate account is missing custom fields: [whisky]"}]"
}
]

挽き目 error_message jsonはエラーオブジェクトの配列になり、オブジェクトには error_code や error_message.

error_code

エラーコードは通常共有に固有のものですが、ビデオ上のPATCH（更新）操作によって返される有効なエラーコードのいずれかである可能性があります。 共有固有のエラーコードを下の表に示します。

メディア共有エラーコード
error_code error_message 原因となる
MISSING_CUSTOM_FIELDS Affiliate account is missing custom fields: [field1, field2]} マスターが設定しました enforce_custom_fields 〜へ true共有されるビデオは、アフィリエイトアカウントに存在しないフィールドを参照します。
CONFLICT Affiliate account is not configured for geo restriction. マスターのenforce_geoが true そのアカウントは地域フィルタリング用に設定されていますが、アフィリエイトアカウントは地域フィルタリング用に設定されていません。
SHARING_DISABLED Affiliate account is not enabled for media sharing. マスターは、メディア共有を有効にしていないアフィリエイトとビデオを共有しようとしました。
NOT_FOUND Resource does not exist. 何かが欠けていました。 通常、これはマスタービデオがなくなったことを意味します。 これは、共有要求の直後にマスタービデオが削除された場合に発生する可能性があります。 これは、マスターアカウントにまだチャネルがないか、アフィリエイトアカウントが存在しないか、チャネルの有効なメンバーではないことも意味します。 これは、移行中に何か問題が発生した場合に発生する可能性があります CMS API メディア共有。
UNKNOWN Sharing failed with an unknown error, error_id INSERT-UNIQUE-ERROR-ID 予期せぬことが起こった。 error_messageには、原因となる原因を特定するためにdevopsが使用できるエラーIDが含まれます。 共有試行を再試行することは多くの場合有効です。
VALIDATION_ERROR {field}: ILLEGAL_VALUE {field}が "economics"の場合、マスタービデオは "AD SUPPORTED"ですが、アフィリエイトアカウントには広告が有効になっていないという問題があります。 また、アカウントが広告用に無効になっているときに、（共有されていない動画の）動画メタデータでサポートされている広告を広告に設定した場合にも発生します。 {field}が別のものであれば、調査が必要です。
REFERENCE_ID_IN_USE Reference id whatever-you-used is already in use. 参照IDはマスターアカウントで有効でしたが、既にアフィリエイトアカウントで使用されています。
