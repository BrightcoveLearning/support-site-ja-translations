メディアモジュール

IE 11による画像キャプチャ

ビデオの縦横比が11：1または1：4の場合、IE 3を使用した画像キャプチャは機能しません。

プレビュープレーヤー

LiveHLSのみのリモートアセット動画は、プレビュープレーヤーでは再生できません。

ダブルバイト文字を含むソースファイル名

2バイト文字を含むソースファイル名は、メディアモジュールで再トランスコードした後に文字化けします。

メディアの共有

大量の動画を共有する場合、 Brightcove 一度に最大2ページの動画を共有することをおすすめします。 一度に多数の動画を共有すると、タイムアウトエラーが発生することがあります。

すでにキャプションが含まれている共有されたビデオにキャプションを追加することはサポートされていません。

クリッピングによって作成されたビデオのメディア共有が失敗する Live ストリーム.

    スケジュールされた動画

    から Playback API 10-15分までのカタログキャッシュビデオは、数分（20まで）の間に利用可能になる予定のビデオを要求するプレーヤーは、キャッシュがリフレッシュされるまで再生可能なビデオを取得できないことがあります。

    Playersモジュール

    予圧設定

    IEを使用している場合、Internet Explorerのバグにより、プリロード設定が無視されることがあります。

    これが Internet Explorerのポリシー設定 ドメインが制限されているために有効になっています Brightcove プレーヤーは、Internet Explorer 11にロードされません。 これはIEのバグであり、既知の回避策はありません。 ドメイン限定プレ​​ーヤーをIE 11で使用するには、この設定をオフにします。

    Live 動画

    Brightcove Player 終了したライブイベントの動画ビューをカウントする

    挽き目 Brightcove プレイヤーは空の、またはもはやリトリーブ可能なHLSプレイリストを持たない終了ライブイベントであっても、リモートアセットがロードされるたびにvideo_viewのアカウントを取得します。 回避策： ビデオの非アクティブ化、スケジュール解除、または削除を行います。

    アナリティクス

    広告ブロッカーが原因で分析データが正確でない場合があります

    UBlockなどの広告ブロッカーがクライアントマシンでの分析データの送信をブロックするため、分析データ（UIとAPIの両方から取得）は不正確になる場合があります。

    一部の動画で配信された0バイトを表示するカスタムレポート

    ダイナミック配信の取り込みプロファイルを使用して取り込まれた動画には、カスタムレポートに配信された0バイトが表示されます。

    複数のフィルタを使用したカスタムレポート

    複数のフィルタを使用する場合は、AND演算が使用されます。 たとえば、2つのフィルタを追加すると、 ビデオタグ や プレイヤー、指定されたタグを持つ動画のみ や 指定されたプレーヤーで表示されます。

    「ビデオタグによるフィルタリング」機能を使用したカスタムレポート

    カスタムレポートを作成するときは、[ビデオタグでフィルタリング]機能を使用するときに制限があります。 その特定のタグを持つあなたの図書館の2,000ビデオまでしか考慮しません。 特定のタグを持つ動画の量がそれより多い場合は、タグを含むフィルタリングされていないレポートを取得し、スプレッドシートソフトウェアを使用してローカルでフィルタリングを実行することをおすすめします。

    「その他/第三者」 パフォーマンスレポートに表示されます

    あなたのビデオではない Video Cloud ライブラリは、 Video Cloud プレーヤー（例えば、リモートアセット）、 その他/サードパーティ 関連するビデオIDなしで表示されます。

    更新された動画名がアナリティクスに反映されない

    アナリティクスで動画のタイトルについて覚えているのは、プレイヤーが再生ビーコンの一部としてアナリティクスに動画を送信したときだけです。 動画タイトルが変更された場合、動画再生が完了するまでアナリティクスは新しい動画タイトルを持ちません。 これにより、動画名が更新され、再生が記録されていない場合、古い動画名がアナリティクスのレポートに反映されます。 3月より前の期間のデータ2013はこの問題に苦しんでおり、数だけのメタデータを表示します。 たとえば、ビデオタイトルが1230123012として表示されることがあります。 これは、この期間のデータが古い分析システムで取得されたためです。

    ???????を表示するビデオ名とプレーヤー名 AnalyticsのUI

    10月の2013では、メタデータ（ビデオ名とプレイヤー名）は?????の束のように見えるかもしれません。 この問題は、メタデータが2バイト文字で、10月の2013でビデオビューを持つパブリッシャに影響します。 この問題は、アナリティクスのUIで表示するメタデータをキャプチャする方法に関連していました。 11月の2013から、アナリティクスシステムには、イベントメトリックとともに送信されたメタデータが破損しないようにするための追加の安全対策が用意されています。

    タイムゾーンのレポート

    レポートのタイムゾーンは、レポートデータの日の境界を計算するために使用されます。 レポートのタイムゾーン設定の変更は、今後のデータにのみ影響し、変更は遡及的に適用されません。 この設定を変更すると、変更が適用された日の境界を確認するときに、データにフラットスポットまたはスパイクが表示される場合があります。 この設定の更新はすぐには有効にならず、システムが変更を処理するまで、分析データは引き続き古い設定を参照する場合があります。

    エンゲージメントデータは毎日の粒度で保存されます

    エンゲージメントデータは、UTC時間を使用して毎日の細かい単位で保存されます。また、2のフルデ日を含むこともあります。Audience エンゲージメントレポートのx％テーブルに表示されます。

    Internet Explorerのサポート

    AnalyticsモジュールにはInternet Explorer 11以降が必要です。

    デバイスメーカの "その他の"結果の数が多い

    場合によっては、デバイスメーカの "その他"の結果が多数表示されることがあります。これは、データコレクタが user_agent 分析データを使用して配信されます。 私たちがこれで特定したもっとも一般的な原因は、Instagramによって作成されたカスタムのuser_agent文字列ですが、おそらく他にもあります。

    Audience

    Audience アクティビティには、監視された割合：0および監視された時間：0が表示されます。

    ビデオが再生されていないとき、またはビデオが最初のビーコンを送信する前に、インタラクティブイベントが発生することがあります。 これにより、インタラクティビティはあるがビデオアクティビティがないビューイベントが記録される可能性があります（0％視聴、0秒視聴）。

    Internet Explorerのサポート

    Audience リードフォームにはInternet Explorer 11以降が必要です。

    Social

    ゴーストブラウザのプラグインが Social モジュール

    Ghosteryブラウザプラグインは、 Social モジュールが正しくロードされないようにします。

    回避策： StudioをGhosteryのホワイトリストに追加します。

    モジュールのアップロード/動的な取り込み

    デバイスの時計が実際の時刻と一致しない場合、アップロードモジュールによるアップロードが失敗する

    アップロードは一定時間許可されます。 あなたの時計の時間が実際の時間から15分以上変化した場合、アップロード時にエラーが発生します。

    動的な摂取による再トランスコードは、動画の有効期限を更新します

    動的な摂取を使用して動画を再トランスコードすると、動画の有効期限が現在の日付に更新されます。 有効期限までに注文されたスマートプレイリストを使用すると、プレイリスト内の動画の順序に影響します。

    ファイル名

    ビデオファイル名（拡張子を含む）は、120の1バイト（60 2バイト）文字を超えてはいけません。 そうした場合、ビデオは正常に取り込まれますが、後で再トランスコードすることはできません。

    Brightcove Player

    チェック Brightcove Player リリースに関して 最近のリリースで過去の既知の問題が修正されているかどうかを確認します。

    コンソールの警告

    ポスターまたはサムネイル画像のない動画

    サムネイル/ポスター画像がないビデオは、読み込まれたときにプレーヤーにエラーが表示されることがあります。 これは、モバイルSDKプレーヤーに影響を与え、プレーヤーをクラッシュさせる可能性があります。

    再生が一時停止されると、Internet Explorerは常に再生レートを1.0にリセットします。

    音声のみのコンテンツのIE11画像は表示されません

    IE11でオーディオコンテンツのみを再生すると、ビデオ静止画は表示されず、黒い画面のみが表示されます。 画像は次のコードで保持することができます：

    .vjs-has-started .vjs-poster {
display: block;
}

    Safariでの自動アドバンスプレイリスト

    Safariの場合、プレイリストが 目に見える、プレイヤーには pre-load 設定は none 、次のビデオの自動再生が黙って失敗します。

    コンテキストメニュー

    プレーヤーの右端または下端に近いビデオを右クリックすると、プレーヤーのコンテキストメニューがオフスクリーンで表示されます。

    同じplayer_idを持つ異なるサイズの複数のプレーヤーをページ上で使用する

    複数のプレイヤーが同じIDを1ページに持ち、異なるサイズを内部で指定している場合 <video> タグを使用して width や height 最後のプレイヤーのプレーヤーCSSがすべてのプレーヤーに適用されます。

    FastClick.jsが正しく処理されない

    FastClick.jsを使用すると、カスタムコントロールバー内の誤ったイベント処理が発生します。 これにより、モバイルデバイスでコントロールバーを使用できなくなる可能性があります。

    ChromeとFirefoxで動画が灰色に表示される

    ChromeとFirefoxを使用している場合、 Brightcove Player 灰色の色で表示されることがあります。 これは、ハードウェアアクセラレーションやNVIDIAドライバの設定が原因である可能性があります。

    回避策： NVIDIAコントロールパネルを開きます。 下 動画 select ビデオのカラー設定を調整する。 下 どのように色を調整するのですか select NVIDIAの設定で。 下 高度な 確認してください ダイナミックレンジ is フル（0-255） Studio上ではサポートされていません。 限定（16-235）. ダイナミックコントラスト強調 チェックしないでください。 この問題は、 Google Chromeヘルプフォーラム.

    エミュレータの使用

    Chromeデバイスモードやその他のエミュレータを使用して動画再生をテストしている場合でも、エミュレータは実際のデバイスの動作を正確に表現できません。 開発中にエミュレータを初期テストに使用することはできますが、実際のデバイスを使用して正確な結果を得ることがベストプラクティスです。

    Chromeデバイスモードで再生をテストすると、次のメッセージが表示されることがあります。「デバイスモードでChromeを使用すると、そのデバイスのビューポートとユーザーエージェント文字列がChromeでレンダリングされます。実際のデバイスの動作を正確に表現できません。 "

    使い方 data-setup

    あなたは使用しないでください data-setup 　 Brightcove Player。 あなたは data-setup APIドキュメントでは、これは、ドキュメンテーションがVideo.jsプレーヤーのソースコードから直接生成され、その属性を video タグを付けてください。 この属性は設定情報をプレーヤーに送信しますが、 Brightcove Player 別の方法を使用してこのタスクを実行します。 data-setup 信頼できない。

    プロトコル対応のソース選択とDASH

    プロトコル対応のソース選択は、DASHコンテンツを使用する場合は使用できません。 これは、HLSおよびMP4コンテンツでのみ利用可能です

    Safariでiframeプレーヤーの実装を使用しているときにコンソールエラーがスローされる

    iframeの実装を使用する場合 Brightcove Player Safariでコンソールに次のエラーメッセージが表示されます。 原点 "http://players.brightcove.net"が原点 "＃DOMAIN NAME＃"のフレームにアクセスするのをブロックしました。 プロトコル、ドメイン、およびポートは一致する必要があります。 報告されたエラーは再生に影響しません。

    Safariのキャプションメニューに「不明」が表示されます。

    これはSafari / Appleの制限です。 新しいバージョンの Brightcove Player Safariに組み込まれたネイティブ字幕機能を使用します。これが標準的な動作です。 これはAppleによって文書化されています： https://developer.apple.com/library/ios/qa/qa1801/_index.html

    フラッシュ以外の環境でのフルスクリーン

    Flashが無効になっているか、FlashベースのHLSが無効になっていて、プレーヤが style の属性 video 一部のブラウザでは全画面表示が機能しないことがあります。

    Social 分担

    デスクトップでは、ソーシャル共有ボタンは広告中に表示されません。

    HTTPS

    ほとんどのデスクトップブラウザでは、 Brightcove Player マニフェストとビデオセグメントの両方がHTTPS接続を介して提供されている場合にのみHTTPS WebサイトでHLSを再生します。 これは、非SSLコンテンツをより厳しく制限するいくつかのブラウザの最近の変更によるものです。 これは、デスクトップコンピュータのChrome、Firefox、Internet Explorerのユーザーに影響します。 Safariユーザーやモバイルブラウザには影響しません。また、MP4レンディションの再生には影響しません。

    我々は、この制限に対処する過程にある Video Cloud管理資産; 独自のCDNとトランスコード（リモートアセット）を管理する場合は、マニフェストコンテンツとビデオコンテンツの両方のHTTPS配信をサポートするようにCDNを設定する必要があります。

    全画面表示

    フルスクリーンAPIをサポートする新しいブラウザでは、フルスクリーンに切り替えるときにプレーヤが100％にスケールされるようにページ内CSSルールを適用する必要があります。 それ以外の場合、プレーヤーは全画面表示の元のサイズで表示されます。 詳細については、 全画面表示 話題 プレーヤーのサイズを決める の資料をご参照ください。

    IE10以前では、フルスクリーンAPIをサポートしていないため、新しいウィンドウが開きますが、プレーヤーはウィンドウを埋めるためのサイズになりません。 これは、プレーヤーのサイズを変更するスタイルが適用されていないためです。 ウィンドウをスクロールすることはできないので、Webサイトのセクションだけが表示され、プレーヤーはまったく表示されません。

    複数のビデオがHapYakの章で1つのページに公開されています

    複数のビデオがHapYakの章で1つのページに公開されている場合、不適切な章が表示される可能性があります。

    回避方法

    このスクリプトを使用します。 https://github.com/brightcove/gallery-in-page-snippets/blob/master/snippets/hapyakFix.js。 ただし、Flashの再生（IE11 / Win7）ではこの問題を回避できないため、IE5 / Win11の場合はHTML7フォールバックを実装する必要があります。

    Akamai HDおよびHDS

    Brightcove Player Akamai HDまたはHDS配信方法によるビデオ配信はサポートしていません。

    エラー： d.offは関数ではありません

    MP4やHLSなどの異なるフォーマット間で切り替えるときに、既知のバグが原因で発生するこのエラーは、プレーヤーで発生する可能性があります。 バグが修正されるまで、問題の原因となっているコードを再試行できます。 次のコードは、アプリのエラーを修正した例です。

    try {
myPlayer.catalog.load(video);
} catch (e) {
myPlayer.catalog.load(video);
}
myPlayer.play();

    HLSビデオ期間

    コントロールバーに表示される時間が最初の表示から変更される可能性があります。 Once すべてのHLSセグメントがロードされ、期間が更新される可能性があります。

    RequireJS

    RequireJSを使用している場合は、 bc() メソッドを使用してプレーヤをインスタンス化します。 このメソッドを使用する必要性は、プレーヤーのアセットがどのようにロードされるかによって決まります。 を参照してください RequireJSと Brightcove Player 詳細はドキュメント。

    referrer_urlとdescription_url

    挽き目 referrer_url iOSとAndroid搭載端末の値が異なる可能性があります。 このため、 description_url 代わりに値。 この値は、すべてのプラットフォームおよびデバイスで一貫しています。

    Safariのループ属性

    挽き目 ループ 属性がSafariで正しく機能しません。 詳細なソリューションを使用することができます。 Brightcove Player サンプル：ビデオループの作成 の資料をご参照ください。

    Safariのキャプション

    Safariはキャプションのスタイル設定にネイティブ機能を使用するため、 キャプション設定 メニューオプションは、 CC Safariブラウザのメニュー（iOSとmacOSの両方）。

    globalという名前のHTML要素IDを使用する

    あなたは、あなたの高度な（ページはめ込みの）プレーヤー実装コードを親に入れ子にしてはいけません <div> タグに id 値を割り当てた global 。 これは Brightcove Player.

    Playerバージョン5アイコンの問題
