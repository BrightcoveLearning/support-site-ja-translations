このトピックでは、オフライン再生が Brightcove ネイティブプレーヤーSDK。

概要

オフラインでの再生 Brightcove iOSとAndroid向けのネイティブプレーヤーSDKを使用すると、コンテンツを安全に保ちながら、全く新しい方法で視聴者にアプローチすることができます。 当社のSDKは、ダウンロード管理、オフラインカタログ管理、分析、DRM、およびもちろん再生など、複雑なオフライン再生の問題に対する強力なシンプルなソリューションを提供します。

オフライン再生を使用すると、ユーザーはDRMで保護されたビデオコンテンツとクリアな（暗号化されていない）ビデオコンテンツをデバイスにダウンロードし、接続されていないときに見ることができます。

この機能を使用するには、 Video Cloud ダイナミックデリバリーを使用しているお客様

サポートされているバージョン

挽き目 Brightcove ネイティブプレーヤーSDKは、次のバージョンのオフライン再生をサポートしています。

デバイスOSのバージョン

  • iOS 10.0 +、ただし10.3 +をお勧めします
  • アンドロイド5.0 +

Brightcove ネイティブSDKのバージョン

  • iOS向けネイティブSDK 6.0.1 +
  • Android用ネイティブSDK 5.0.3 +

コンテンツ配信

ストリーミングビデオは、 Video Cloud同社のマルチビットレートストリーミング機能は、ソースでの帯域幅を検出し、デバイスに最適な品質を提供します。 オフライン再生機能は、次のストリーミングタイプをサポートしています。

  • Android：HTTP（DASH）による動的アダプティブストリーミング
  • iOS：Apple HTTP Live ストリーミング（HLS）

セキュリティ

ダウンロードされたコンテンツの保護は、オフラインソリューションの最優先事項です。 ザ Brightcove ネイティブプレーヤーSDKは、コンテンツが視聴者のデバイスに安全に保存されるよう、次の機能を提供します。

DRM保護されたコンテンツ

  • DRMで保護されたコンテンツの場合、ネイティブSDKは次のテクノロジを使用します。
    • Android：DASH with： Widevine Modular

    • iOS：HLS with： フェアプレー

  • 動画コンテンツは、コンテンツをダウンロードした端末やアプリでのみ視聴できます。

  • ビデオコンテンツは、特定の日時の有効期限までしか視聴できません。

    ネイティブSDKを使用すると、ユーザーはビデオをレンタルまたは購入できます。 有効期限はレンタルに適用され、動画をダウンロードするためのネイティブSDK APIを使用してDRMライセンスで設定されます。 対照的に、Fastlyトークンで設定された有効期限は、ストリーミングコンテンツに適用されます。

  • クライアントに返された動画URLは、他のユーザーが使用できないように期限が切れます。

すべてのコンテンツ

  • アプリが削除されると、ダウンロードされたすべてのコンテンツが削除されます。
    Androidの場合、これは true デフォルトの保存場所が上書きされない限り。
  • あなたのアプリは、いつでもローカルに保存されたコンテンツを削除できます。

ダウンロード管理

挽き目 Brightcove ネイティブプレーヤーSDKは、コンテンツを視聴者のデバイスにダウンロードする複雑さを処理し、ネイティブアプリケーションのプロセスを大幅に簡素化します。

  • パブリッシャーは、ダウンロードするコンテンツのレンディション品質を選択できます。
  • キャプションはオフラインでダウンロードして表示できます。
  • Native SDKでは複数のオーディオトラックがサポートされています。 追加トラックのダウンロードの詳細については、以下を参照してください。
    • Android用ネイティブSDKを使用したオフライン再生用の複数のオーディオトラック
    • iOSアプリケーションデベロッパーガイドオフライン再生.

機能性

ダウンロード管理をサポートするために、以下の機能をアプリケーションに追加することができます：

  • 表示内容
  • ダウンロードサイズを確認する
  • ダウンロード状況を表示
  • ダウンロードキューを管理する
  • ローカルストレージスペースを表示する
  • ダウンロードの一時停止/再開/キャンセル

表示内容

ストリーミングまたはダウンロードに使用できるコンテンツのリストを表示します。

動画を設定すると offline_enabled プロパティへ trueネイティブSDKはこのビデオをダウンロード可能とマークします。

ビデオを準備する

ビデオをダウンロード可能としてマークするには、次の操作が必要です。

  1. アカウントをオフライン再生できるようにするには、アカウントマネージャーにお問い合わせください。

  2. ダウンロード可能にするビデオごとに、次のいずれかを実行してオフラインを有効にします。

    Linuxを使う Video Cloud Studio

    Linuxを使う Video Cloud スタジオ オフライン再生を有効にする.

    Linuxを使う CMS API

    使用することもできます CMS API を設定する offline_enabled 値のフィールドに true.

    curlを使用した例を次に示します。

      curl --header "Authorization: Bearer $oauth_token" --request PATCH --data '
  {"offline_enabled" : true}
  ' https://cms.api.brightcove.com/v1/accounts/your account id/videos/your video id

明確なコンテンツを扱う場合は、 ベストプラクティス コンテンツの表示方法に関するヒントについては、セクションをご覧ください。

ダウンロードサイズを確認する

Brightcove ダウンロードを要求する前に、ダウンロード用のデバイスストレージに十分なスペースがあることを確認することをお勧めします。 ビデオダウンロードの合計サイズの推定値を取得するには、次の手順を実行します。

  • iOSの場合は、 ダウンロードサイズを確認する のセクション iOSアプリデベロッパーガイドオフライン再生.

  • Androidの場合は、 OnDownloadStarted からのコールバック DownloadEventListener。 次に、イベントリスナーを設定する方法の例を示します。

      MediaDownloadable.DownloadEventListener downloadEventListener = new MediaDownloadable.DownloadEventListener() {
  //Your implementation.
  }
  OfflineCatalog catalog = new OfflineCatalog(context, eventEmitter, accountId, policyKey);
  catalog.addDownloadEventListener(downloadEventListener);

ローカルストレージスペースを表示する

現在のアイテムおよびキュー内のアイテムのダウンロードを完了するのに十分なローカルストレージがない場合、ユーザーに通知します。

SDKは、ダウンロードのための十分な領域があるかどうかを判断できるように、推定サイズと進捗状況を提供します。

ダウンロード状況を表示

現在のダウンロードとそのステータスを表示します。

推定された合計サイズ、現在のダウンロード状況、およびダウンロード中の進捗情報の割合は、SDKによって返されます。

Androidデバイスの場合、ダウンロードステータスは通知エリアにも表示されます。 通知のタイトルはビデオタイトルに設定されます。 ダウンロードが一時停止またはキャンセルされた場合、通知は削除されます。 詳細については、 Android用Native SDKを使用したコードスニペット の資料をご参照ください。

ダウンロードキューを管理する

ダウンロードキューを作成し、管理します。

ネイティブSDKでは、複数の動画をダウンロード用にキューに入れることができます。 たとえば、ユーザーはシーズンの複数のエピソードをダウンロードでき、ネイティブSDKは各アセットのダウンロード状況を提供することでプロセスを管理します。

ノート：

  • アプリがiOS 11.0-11.2をターゲットにしている場合、一時停止と再開の動作は複数の同時ダウンロードでは信頼できないため、同時ダウンロードはお勧めしません。 問題はiOS 11.3で修正されました。

ダウンロードの一時停止/再開/キャンセル

ユーザーがコンテンツのダウンロードを一時停止、再開、またはキャンセルできるようにします。

ユーザーが一時停止/再開した場合、またはダウンロードが中断された場合、中断したところからダウンロード機能が再開されます。 たとえば、ダウンロード中にWi-Fi接続を失うと、再接続したときに中断したところから再開します。

ビデオのダウンロードをキャンセルすると、進行状況が消去されます。

ローカルカタログ管理

挽き目 Brightcove ネイティブプレーヤーSDKは、ダウンロードしたローカルコンテンツの管理プロセスを簡素化します。

ローカルカタログ管理をサポートするために、以下の機能をアプリケーションに追加することができます。

  • ダウンロードしたコンテンツを表示する
  • ローカルコンテンツを再生する
  • ローカルコンテンツを削除する

ダウンロードしたコンテンツを表示する

各ショー/シーズンにダウンロードしたすべてのエピソードを表示します。

SDKは、ダウンロードされたすべてのコンテンツのメタデータと再生の詳細を返します。 このメタデータには、ビデオタイトル、説明、サムネイル、使用可能なキャプション、使用可能なオーディオトラックなどが含まれます。

オフライン再生に固有の新しいメタデータプロパティは次のとおりです。

DRM保護されたコンテンツ

  • トークン識別子
  • ライセンス有効期限

すべてのコンテンツ

  • ダウンロード開始時間と終了時間
  • ダウンロードしたサムネイル名とそのファイルのURL
  • ダウンロードしたポスターの名前とそのファイルのURL
  • ビデオバンドルの相対および派生ファイルのURL

次に、特定の実装の結果をフィルタリングできます。

レンタルライセンスの残り時間を確認することができます。

ローカルコンテンツを削除する

ユーザーがローカルストレージからコンテンツを削除できるようにする。

SDKは、メタデータと関連するアセットと共にビデオを削除する方法を公開しています。

コンテンツの再生

ネイティブSDKを使用すると、次の状況でコンテンツを再生できます。

  • ダウンロード中にビデオを再生します。
  • ダウンロードした後にビデオを再生します。 ダウンロードしたコンテンツは、デバイスがオフラインまたはオンラインのときにローカルで再生できます。 オフラインビデオはローカルストレージからしか再生できません。
  • ビデオのオンライン版をストリーミングする場合は、再生サービスから新しいビデオオブジェクトを取得する必要があります。

アナリティクス

オフラインフローに関するデータを測定するためのアナリティクスが導入されました。 詳細については、 ネイティブSDKを使用したオフライン再生に関するアナリティクス の資料をご参照ください。

ノート

ネイティブSDKでサポートされているオフライン再生機能には、次の注意事項が適用されます。

全て

広告

オフライン再生では、クライアント側の広告もサーバー側の広告もサポートされていません。

HLSe

現在、HLSeコンテンツは、オフライン再生ではサポートされていません。 Brightcove Android、iOS、tvOSのネイティブSDK

低消費電力モード

デバイスの低電力モードは、ダウンロードなどの特定の機能に影響する場合があります。 詳細については、次を参照してください。

  • 電源管理Android 9（APIレベル28）
  • iPhoneの低電力モード
  • iPhoneで低電力モードに反応する

共有動画

共有ビデオは、元のビデオが可能な場合にのみ、オフラインで表示するためにダウンロードできます。 このプロパティは元のビデオから継承され、共有ビデオに対してのみ変更できません。

オーディオのみ

ネイティブSDKは、オフライン再生でオーディオのみのコンテンツをサポートします。 それがポスター画像であるかカスタム画面であるかにかかわらず、視聴者が見るものを制御します。

AndroidのネイティブSDKには、 VideoView、およびiOS用のネイティブSDKには、 UIView。 このため、バックグラウンドオーディオはサポートされていません。 これは、ユーザーがアプリをバックグラウンドに送信したときに音声が再生されないことを意味します。

Android

アプリの削除

アプリが削除されると、ダウンロードされたすべてのコンテンツが削除されます。 これは true デフォルトの保存場所が上書きされない限り。

iOS

ダウンロード管理

iOS 13では、Appleはダウンロードの動作を変更しました。 詳細については、 動画ダウンロードとオフライン再生に関するiOSアプリ開発者ガイド ガイド。

この変更により、iOSオフラインダウンロードサンプルアプリは、複数のオーディオトラックを含むビデオを再ダウンロードするように見える場合があります。 これは、ダウンロードの進行状況が各トラックごとに表示されるためです。 そのため、追加のオーディオトラックごとに、プログレスバーがゼロから再び開始されます。

AirPlayを

セキュリティ機能として、オフラインHLSビデオをAirPlayデバイスにストリーミングして再生することはできません。 これはAppleによって確認されています AVFoundation 制限。

よくある質問

以下に、いくつかの一般的な質問に対する回答を示します。

DRM保護されたコンテンツ

有効期限を迂回するようにシステムクロックを変更することはできますか？

Android：絶対有効期限は、ライセンスが有効になるまでの日時を決定します。 これは、WidevineとAndroid DRM層によって適用されます。 理論的には、AndroidのDRM層は、ユーザーが有効期限を迂回するのを防ぐ必要があります。 私たちが見つけた問題は、Android OSのリリースノートに記載されています。

iOS：ライセンスの有効期限はFairPlayによって強制されます。 iOSは時計の変更などのすべてのチェックを処理します。アプリは、ライセンスの有効期限をチェックして、ライセンスの有効期限が切れたことをユーザーに警告するオプションを備えています。 有効期限の切れたビデオを再生し、再生エラーを受け取ることが好ましい場合があります。

アプリがバックグラウンドにある間に、ライセンスとオフライン動画が引き続き読み込まれますか？

Android： はい。 通常、ライセンスはすぐにダウンロードされますが、アプリがバックグラウンドになっている間にライセンスをダウンロードすることもできます。

iOS：ライセンスの読み込みはフォアグラウンドで行われます。 私たちは ライセンスをプリロードする方法。 主なビデオコンテンツのダウンロードは、アプリがバックグラウンドの間に行われる可能性があります。

すべてのコンテンツ

オフラインコンテンツのダウンロードに使用されるビデオ形式は何ですか？

AndroidDRMで保護されたDASHまたは暗号化されていないDASH。

iOS：HLS蒸し。 ダウンロードのビットレートを指定して、特定のバリアントを使用することができます。

ユーザーがオンラインのときにダウンロードしたビデオをローカルで再生するか、ストリーミングしますか？

ダウンロードしたビデオは、オンラインまたはオフラインで再生できますが、ダウンロードしたビデオは常に使用されます。 オンラインバージョンのビデオを再生する場合は、再生サービスから新しいビデオオブジェクトを取得する必要があります。 オフラインビデオはローカルストレージからしか再生できません。

ベストプラクティス

オフライン再生用にアプリを開発する場合は、次のガイドラインを参考にしてください。

一時停止/キャンセルボタンの表示

ビデオのダウンロードが開始されると、一時停止ボタンとキャンセルボタンが表示されます。

アンドロイド： 呼び出しの一環として、一時停止ボタンとキャンセルボタンを表示する必要があります onDownloadStarted のコールバック DownloadEventListener。 詳細については、 オフライン再生サンプルアプリケーション.

iOSの： オフライン再生用のアプリケーションを作成する方法の詳細については、 iOSアプリケーション開発者ガイド.

明確なコンテンツを表示する

アカウントでDRMが有効になっていない場合や、クリアコンテンツで作業している場合は、 Brightcove ネイティブSDKはサーバー側のライセンス要求を行いません。 したがって、明確なコンテンツは、レンタルまたは購入のために利用可能とはみなされません。 ほとんどの場合、明確なコンテンツを表示するには、コンテンツを非表示にすることをお勧めします。 家賃 や 購入 ボタンを1つ表示 ダウンロード ボタンをクリックします。

機能の有効化

課金については、アカウントマネージャーに連絡し、アカウントのオフライン再生機能を有効にしてください。

トラブルシューティング

次のヒントは、ビデオをダウンロードする際の問題を調査するのに役立ちます。

ネットワークが遅いためにユーザーがデバイスで処理できるよりも高いビットレートでアセットをダウンロードしようとすると、エラーが発生する場合があります。

これを回避するには、デバイスの機能を確認するコードを追加し、低速接続のデバイスが高ビットレートのアセットをダウンロードしないようにします。 接続を調整しながら実際のデバイスでテストし、ネットワークトラフィックを表示できます Charles 代理 ツール。

Android

エラー処理の詳細については、 Android用ネイティブSDKのエラーメッセージ 資料。 これには、オフライン再生に関連するエラーメッセージが含まれます。

iOS

エラー処理の詳細については、 iOS向けネイティブSDKでのエラー処理 の資料をご参照ください。

オフライン再生コードがエラーを受け取ると、それをキャッチしてNSErrorをスローします。 根本的なエラーを取得するには、これを試すことができます：

  NSError *underlyingError = initialError.userInfo[NSUnderlyingErrorKey];

詳細については、以下を参照してください オフライン再生用のiOSアプリケーションデベロッパーガイド.
