Live API の資格情報管理

製品（複数可）

Live API

このトピックでは、 Live API のさまざまな種類の資格情報を管理するための概要について説明します。

概要

一緒に使用することができるいくつかの種類の資格があります：

  • FTP、S3、 Video Cloud、もしくは Zencoder にクリップをプッシュするための資格情報
  • ライブストリームのRTMPアドレスを取得した人がそれを再生できないようにするためのRTMP認証情報
  • OAuth2または基本認証等の Live システムからの通知ハンドラに用いられる認証情報

そのための資格証明エンドポイント Live API これらの資格情報を作成して管理することができます。これらの資格情報は、作成時にラベルで参照することができます Live またはVODクリップジョブ。

資格情報のAPI操作の詳細については、 APIリファレンス.

資格情報タイプ

次の表は、現在サポートされている資格情報の種類とその使用方法を示しています。

資格情報タイプ
credential_type 値 説明/使用例
ftp クリップを送信するFTPサイトの資格情報
http httpアドレスが通知を送信するための基本認証資格情報
oauth 通知を送信するアドレスにOAuth2認証の資格が必要
rtmp エンコーダが接続するエンドポイントに認証を追加すると、RTMP URLを持つ人は誰でも接続できます
s3 クリップを送信するS3バケットの資格情報
videocloud あなたのOAuth資格情報 Video Cloud クリップを送信するアカウント（Dynamic Ingestに必要な権限を含む必要があります）
zencoder あなたのための認証 Zencoder クリップを送るアカウント

資格情報の操作

次の表は、資格情報を管理するためのエンドポイントを示しています。

資格証明エンドポイント
エンドポイント 方法 説明
/v1/credentials GET アカウントのすべての資格情報を一覧表示する
/v1/credentials POST アカウントの新しい認証情報を作成する
/v1/credentials/{credential_id} PUT アカウントの資格情報を更新する
/v1/credentials/{credential_id} DELETE アカウントの認証情報を削除する

資格情報フィールド

次の表は、資格情報のフィールドを示しています。

資格情報フィールド
Field Type Required Allowed Values Default Description
credential_default_for_type boolean no true | false false Want this credential to be default for the user for this type of credentials?
credential_label string yes     User identifiable name for the credentials
credential_type enum yes ftp | http | oauth | rtmp | s3 | videocloud | zencoder   The type of credential
credential_public string yes     The username, public key, or client id, depending on the credential type
credential_private string yes     The password, private key, or client secret, depending on the credential type
user_id string no   (credentials are available for all account users) user_id (GUID) for which the credentials are being created
