このトピックでは一般的な Insomnia HTTP クライアントを設定して、Brightcove RESTful APIでリクエストを実施する方法を学びます。

概要

プラットフォームAPIドキュメントの多くの例で使用されているcurlステートメントやコマンドラインは難しく、わかりづらい部分があります。 そのため、RESTベースのサービスには多くの Brightcove API を含む、HTTPリクエストを送信するためのツールが数多くあります。 このドキュメントでは一般的なInsomnia アプリツールを使用する方法について説明します。

Insomniaをインストールする。

Insomniaをhttps://insomnia.restから取得します。Insomnia は Mac、Linux、または Windows システムにインストールできます。

Insomnia はBrightcove の RESTful API のいずれかにリクエストを行うために使用できます。APIのほとんどは認証にOAuthXNUMXを使用しており、それらは私たちがこのドキュメントでデモを行なっているものです。しかしながらこれらは適切なヘッダーを設定するだけで、OAuth を使用しない API にも使用できます。

異なる認証方法を使用したリクエストではヘッダーは次のようになります。

OAuth API （クレデンシャルリクエストを作成する のみ)
  Authorization: BC_TOKEN YOUR_BC_TOKEN

あなたのBC_TOKENを取得する方法。

Playback API
  BCOV-Policy: YOUR_POLICY_KEY
Live API
  X-API-KEY: YOUR_API_KEY
Zencoder API
  Zencoder-Api-Key: YOUR_API_KEY
Once API
  X-BC-ONCE-API-KEY: YOUR_API_KEY

クライアントの資格情報を取得する

ほとんどの Brightcove APIを使用するには、使用するアカウントとAPIのクライアント資格情報が必要です。API 認証クレデンシャルの管理を参照の上、Studioのクライアント資格情報を取得します。以下の手順ではInsomniaを使用してPlayer Management APIリクエストを実施しているため、資格情報には少なくとも次の権限が必要です。

  • Players: Read/Write

より多くの追加のアクセス許可を追加して、より広範なAPI要求に使用できる資格情報を取得することができます。 また必要に応じて複数のアカウントで動作する資格情報を取得することもできます。

クライアントの資格情報を生成するには、Studioの管理者権限が必要です。または、管理者権限を持つ組織内のユーザーに資格情報を生成させる必要があります。

環境変数

Insomnia環境を使うと、さまざまな API の OAuth 認証情報を保存したり、アカウントID、動画、プレーヤーIDなどのその他の情報を保存できます。

以下はclient_id, client_secret と account_idを使って環境変数を作成および使用するための手順です。

  2. . 環境を管理する ダイアログで、 + 横のサイン サブ環境 ドロップダウンメニューを展開し、 環境 新しい環境を作ります：
  3. 新しい環境ダブルクリックし、名前を Player Management API （または任意の名称）へ変更します。
  4. 環境JSONの中括弧の中に、次のキーと値のペアを追加します。
    • "account_id""アカウントID"
