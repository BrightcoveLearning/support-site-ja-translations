このトピックでは、ライブストリームジョブ用にサーバーサイド広告挿入（SSAI）を設定する方法を示します。

概要

SSAI 指定した時間にライブストリーミングイベント中に広告を表示できます。 主な機能は次のとおりです。

  • エンコーダから送信されたキューポイントを使用して広告を挿入するか、またはを使用してインスタントキューポイントを作成します。 Live API
  • 未使用の広告時間を埋めるために「スレート」アセットを取り込む
  • サーバー側でライブストリームにつながれる広告で広告ブロッカーを避ける
使用する前に SSAIあなたのアカウントで有効にする必要があります。 詳細については、アカウントマネージャーにお問い合わせください。

作成するには Live サーバーサイド広告でストリーミングするには、次の手順に従います。

  1. についての一般的な情報を確認してください。 Live API
  2. ライブ広告設定を作成する。 あなたもこれを行うことができます Video Cloud Studio。 を使用して広告設定を管理する方法の詳細については、以下のセクションをご覧ください。 Live API.
  3. スレート資産を作成する。 あなたがUIを好めば、あなたはすることができます スタジオでスレートを管理する.
  4. 任意： キューポイントと広告ビーコンを挿入する.
  5. 今、あなたは準備ができています 作ります Live を使用して Live API。 Studioを使用したい場合は、 サーバーサイド広告を Live モジュール

カスタムヘッダー、広告設定変数、DFPと広告マクロの使用の詳細については、このドキュメントの残りの部分を参照してください。

一般的な情報

次の情報はすべてに関係します Live API SSAI要求を使って。

ベース URL

そのベースURL Live API SSAIの場合：

    https://api.bcovlive.io/v1/ssai

認証

リクエストの認証にはAPIキーを持つヘッダが必要です。

    X-API-KEY: your API KEY

アカウントマネージャに連絡して、アカウントに関連付けられているAPIキーを入手してください。

スレートの管理

スレートは、未使用の広告時間を埋めるために使用される独自の資産です。 あなたはスレートを使って、 "be right back"メッセージや好きなコンテンツを提供することができます。

以下は、スレート資産を追加および管理するためのAPIリクエストの詳細です。

スレート資産を追加

新しいスレートメディアソースアセットを取り込むには、 POST リクエスト：

方法 POST
URL https://api.bcovlive.io/v1/ssai/slates
ヘッダ X-API-KEY: your API KEY

サンプルリクエストボディ

    {
      "source_url": "https://somesourceasset.com/slate-to-ingest.mp4",
      "account_id": "Account_ID [Optional]",
      "source_description": "User identifiable description for the slate  [Optional]"
    }
. account_id ここにある Live アカウントID。 省略した場合は、要求元ユーザーのアカウントIDが使用されます。

サンプルレスポンス

    {
      "media_source_asset_id": "New_UUID",
      "account_id": "Account_ID",
      "media_source_asset_default": false,
      "media_source_asset_type": "slate",
      "media_source_asset_url": "https://somesourceasset.com/slate-to-ingest.mp4",
      "media_source_asset_status": "transcoding"
      "media_source_asset_description": "User identifiable description for the slate"
    }
注： media_source_asset_defaultを変更することは、サポートされていません。

DFPに関する注意事項

DFPから広告を取得する場合は、問題を回避するために注意する必要があることをここに示します。

広告タグ

広告タグを作成するとき Live、適切なガイドラインに従っていることを確認してください。 /live 。 DFPドキュメントを参照してください マスタービデオタグを手動で作成する 完全な詳細については。

並行性

並行性が高いと思われる場合は、DFPアカウントチームに相談することをおすすめします。

単数/複数の広告レスポンス

. singleadresponse と multiadresponse パラメータは現在では使用されていません SSAI.

Live SSAI 1回の広告サーバー呼び出しのみを行い、応答には1広告あたり5リダイレクトの制限で必要な広告ラッパーに従うことを除いて、休憩のためのすべての広告が含まれることを期待します。 次の広告レスポンス形式が受け入れられ、次のように解析されます。

  • VAST - 1つのレスポンス内の単一のレスポンスまたは広告の1つのポッド
  • DFP広告ルール - プレロール、ミッドロール、ポストロールで定義された広告を含むレスポンスに使用可能なすべての広告を集計します
  • スマートXML - プレロール、ミッドロール、ポストロールの定義済み広告を含むレスポンスに使用可能なすべての広告を集約します

広告リクエストのカスタムヘッダー

. SSAI platformは、Ad呼び出しとバックエンドプラットフォームで使用されるすべてのビーコンとともにカスタムヘッダーを渡すことができます。 VideoPlazaなどの一部の広告サーバーではカスタムヘッダーが必要です。

カスタムヘッダーは、一連のキーと値のペアとして指定されます。 ad_configuration_headers の一部であるオブジェクト application_ad_configuration （参照 広告設定を作成する セクション）。

備考

  • 標準ヘッダーは、次のようにデフォルトで処理されます。
    • X-Forwarded-For
    • X-Device-User-Agent
  • ヘッダー値には、 広告構成変数
  • ヘッダー値は、静的な文字列にすることもできます
ヘッダーは、広告ラッパーやインプレッション/四分円を含むすべてのダウンストリームリクエストに渡されるので、注意して使用する必要があります。

広告マクロを使用して広告をターゲティングする

ときにあなたを 広告設定を作成する広告タグは通常次のようになります。

      https://ad-provider-host.com/path/to/ad-handler?ip={{client.ipaddress}}&
      num={{random.int32}}&ses={{session.session_id}}&IDFA={{deviceid}}&
      sitesection={{sitesection}}&breakid={{breakid}}&breaktheme={{breaktheme}}

二重中括弧内の項目は、広告マクロとも呼ばれる変数です。 Brightcove Live 存在する場合は、広告リクエストを送信するときに値で置き換えられます。

役立つ変数は広告サーバーによって異なります。

広告マクロの値は次の方法で指定できます。

  • ヘッダ情報を使う
  • URLを追加する
  • URLを動的に追加する
  • 手動キューポイントの使用

ヘッダ情報を使う

ヘッダー情報は、 広告設定変数 上記のセクションは、任意の要求に利用可能です。 広告設定で適切なキー名を使用して必要な変数を指定するだけです。

URLを追加する

次のように、追加のセッション値をライブストリームのURLに追加できます。

      http://playback.bcovlive.io/e058d9f2737e18/us-west-2/NA/
      playlist.m3u8?deviceid=123456&sitesection=services"

URLを動的に追加する

Javascriptを使用してURLを動的に追加できます。 Brightcove Player API：

      <!DOCTYPE html>
      <body>
        <video
        id="myPlayerID"
        data-video-id="5975635167001"
        data-account="3737230870001"
        data-player="tIG7lVKBm"
        data-embed="default"
        data-application-id
        class="video-js"
        controls
        width="640"
        height="360"></video>
        <script src="//players.brightcove.net/3737230870001/tIG7lVKBm_default/index.min.js"></script>

        <script type="text/javascript">
        var player = videojs("myPlayerID");

        player.one("loadstart", function(){
          var sourceUrl = player.currentSource();
          console.log(sourceUrl);
          var liveUrlWithParams = sourceUrl.src + "?player_width={width}&player_height={height}&deviceid={deviceid}";

          player.src([{
            "type": "application/vnd.apple.mpegurl",
            "src": liveUrlWithParams
          }]);
        });
      </script>

      </body>

この例のキー名は、上に示した広告タグの変数名に対応しています。

広告設定変数

ビーコンとも呼ばれる広告構成変数は、広告構成を管理するリクエストで使用できます。 詳細については、 Live API：SSAIによるキューポイントとアドビーコン をご参照ください。

手動キューポイントの使用

特定の広告ブレークの値は、手動キューポイント挿入要求と共に送信できます。 詳しくは、 Live API：キューポイントとアドビーコン をご参照ください。

既知の問題

  • SSAIでは、ライブストリーミングビデオにオーディオトラックが必要です。
  • 返されたVASTに同じ広告IDが設定されている場合、広告URLは動的変数を使用して一意のURLにしても、広告コンテンツをリクエストしません。 これは 次の例は正しくありません。 広告にDFPを使用している場合は適用されます。
  • DFPでは、同じ広告IDを使用できますが、それでも異なるクリエイティブIDが必要です。つまり、MediaFileを単純に交換することはできません。

  • 広告ブレークの持続時間が広告URLのMAX期間（min_ad_duration = 0＆max_ad_duration = 30000）未満の場合、広告が広告ブレークよりも長く返された場合、広告は再生されません。

  • VPAIDまたはクリック可能な広告はサポートされていません Brightcove Live SSAI.

  • 広告ブレークの残り時間がストリームのセグメント秒よりも短く、スレートが表示されると、スレートはセグメント期間で表示され、実際の広告ブレークは予想よりも長くなります。

  • 広告がGoogleのシステムで初めて表示されるときは、トランスコードされて配信の準備が整うまで再生されません。
  • VMAP Live SSAIは現在サポートされていません。
