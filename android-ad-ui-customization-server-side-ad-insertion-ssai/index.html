このトピックでは、サーバーサイド広告の再生中に表示されるUI要素を、SSAIプラグインを使用して管理およびカスタマイズする方法を学習します。 Brightcove Android用ネイティブSDK

概要

サーバー側広告挿入（SSAI）を使用する場合、 Brightcove ネイティブSDK for Androidでは、広告UI要素をカスタマイズできます。 SSAIの詳細については、次を参照してください。

  • ネイティブプレーヤーSDKを使用したサーバーサイド広告の実装
  • Android用ネイティブSDK用のSSAIプラグイン

広告UIは、2つの主要コンポーネントで構成されています。

  • 広告オーバーレイ（A）
  • 広告メディアコントローラー（B）

広告オーバーレイ

広告オーバーレイは、次のコンポーネントで構成されています。

  • 広告ポッドの合計継続時間のカウントダウン（A）
  • 詳細ビュー（B）
  • 広告ビューをスキップ（C）
[広告のスキップ]ボタンは、VODサーバー側広告挿入（SSAI）でのみサポートされています。 現在サポートされていません Live SSAI。

広告メディアコントローラー

広告メディアコントローラーは、左から右に次のコンポーネントで構成されています。

  • 再生ボタン（A）
  • 広告番号のカウントダウン（B）
  • 単一の広告期間カウントダウン（C）
  • 全画面ボタン（D）

広告UIコンポーネントの管理

ビューから広告UIコンポーネントを有効または無効にするには、次を参照してください。

  • VASTドキュメントの変更
  • 広告構成の指定

VASTドキュメントの変更

挽き目 もっと知りたい や 広告をスキップ ビューは、VASTドキュメントの広告定義に基づいて表示または非表示にできます。

挽き目 もっと知りたい ビューは、リニアクリエイティブに ClickThrough 以下に示すURL：

 <VAST xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Ad id="blue-01-5s">
    <InLine>
      ...
      <Creatives>
        <Creative id="creative-01-5s">
          <Linear>
            ...
            <VideoClicks>
              <ClickThrough id="clickthrough">https://www.brightcove.com/en/</ClickThrough>
              <ClickTracking id="...”>...</ClickTracking>
            </VideoClicks>
          </Linear>
        </Creative>
      </Creatives>
    </InLine>
  </Ad>
</VAST>

挽き目 広告をスキップ ビューは有効なものがある場合にのみ表示されます skipoffset リニアクリエイティブの値：

 <VAST xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Ad id="blue-01-5s">
    <InLine>
      ...
      <Creatives>
        <Creative id="creative-01-5s">
          <Linear skipoffset="00:00:03">
            ...
          </Linear>
        </Creative>
      </Creatives>
    </InLine>
  </Ad>
</VAST>

広告構成の指定

使用時 プレーヤーの機能強化 Live SSAI、内のコンポーネントを有効または無効にすることができます application_ad_configuration 広告設定の。 の client_options オブジェクトを使用すると、広告UIコンポーネントをオンまたはオフにできます。

広告設定を使用した広告UIの変更は、以下にのみ適用されます Live SSAI。
{
  "application_ad_configuration": {
	"ad_configuration_description": "$YOUR_DESCRIPTION",
	"ad_configuration_expected_response_type": "Vast",
	"ad_configuration_headers_for_impressions": false,
	"ad_configuration_strategy": "SingleAdResponse",
	"ad_configuration_transforms": [],
	"ad_configuration_url_format": "$YOUR_AD_SERVER",
     "ad_configuration_client_sdk_enabled": true,
     "client_options": {
        "show_ad_break_remaining_time": true / false,
        "show_ad_remaining_time": true / false,
        "show_number_of_remaining_ads": true / false,
        "client_only_tracking": true / false
     }
  },
  "application_description": "$YOUR_DESCRIPTION"
}
いずれかを使用している場合 client_options コンポーネント、あなたが持っている必要があります ad_configuration_client_sdk_enabled に設定 true.

client_optionsオブジェクトを使用して、次のコンポーネントを有効/無効にします。

  • Show_ad_break_remaining_time：広告ポッドの合計継続時間のカウントダウン
    • 例： 動画は15秒後に再開されます
  • Show_ad_remaining_time：単一の広告期間カウントダウン
    • 例： （0：15）
  • Show_number_of_remaining_ads：広告数のカウントダウン
    • 例： 広告1/3, 広告2/3などです。

広告オーバーレイのカスタマイズ

広告オーバーレイに関連付けられたビューとデフォルトのテキストをカスタマイズできます。

ビューの変更

広告オーバーレイをカスタマイズするには、バージョンを置き換えることができます ssai_ad_overlay.xml アプリのレイアウトファイル 解像度 フォルダ。 このファイルには、次に示すように、いくつかの異なる独立したレイアウトが含まれています。

たとえば、Ad Podの継続時間ビューをオーバーライドするには、 ssai_ad_pod_duration_countdown_view.xml ファイル。 また、広告オーバーレイビュー全体を上書きすることもできます。 ssai_ad_overlay.xml ファイル。

このレイアウト構造により、古いSSAIプラグインバージョンとの下位互換性を維持しながら、特定のコンポーネントをオーバーライドできます。

Android Studioを使用して広告オーバーレイファイルを調べます。 ネイティブSDK for Androidのバージョン6.9.0では、SSAIプラグインの ssai_ad_overlay.xml 次のようになります。

<?xml version="1.0" encoding="utf-8"?>
<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:background="@color/white">

   <include layout="@layout/ssai_ad_countdown_and_learn_more_overlay"/>
   <include layout="@layout/ssai_ad_skip_overlay"/>
</FrameLayout>

コンポーネントのxmlファイルを置き換えるときは、 TextView 意図しない結果を避けるために、元のビューIDが指定されています。 詳細については、次の表を参照してください

コンポーネント名 コンポーネントID コンポーネントビュー
広告ポッドの継続時間ビュー text_ad_countdown TextView
詳細を見る text_ad_learn_more TextView
広告ビューをスキップ text_ad_skip TextView

デフォルトのテキストを変更する

すべての広告オーバーレイコンポーネントは TextViewsアプリで定義されている string.xml ファイル。 表示テキストを変更するには、作成して上書きします string or plurals Native SDK for Androidで使用されるものと同じ文字列IDを使用するアイテム。

挽き目 広告ポッドの期間 ビューは次の文字列を使用します。

<plurals name="ssai_message_ad_break_duration_countdown">
   <item quantity="one">Your video will resume in\n%1$d second</item>
   <item quantity="other">Your video will resume in\n%1$d seconds</item>
</plurals>
<string name="ad_info_now_text">Your video will resume now</string>
<string name="ad_buffering_text">Your ad is buffering …</string>

挽き目 もっと知りたい ビューは次の文字列を使用します。

<string name="ssai_message_learn_more">Learn More >> </string>

挽き目 広告をスキップ ビューは次の文字列を使用します。

<plurals name="you_can_skip_text">
   <item quantity="one">You can skip this ad in\n%d second</item>
   <item quantity="other">You can skip this ad in\n%d seconds</item>
</plurals>
<string name="skip_text">Skip Ad</string>

Ad Media Controllerのカスタマイズ

Ad Media Controllerに関連付けられたビューとデフォルトのテキストをカスタマイズできます。

ビューの変更

Ad Media Controllerをカスタマイズするには、バージョンを置き換えることができます default_ssai_ad_media_controller.xml アプリのレイアウトファイル 解像度 フォルダにコピーします。

これは、SSAIプラグインで使用されるデフォルトのレイアウトをオーバーライドします。 広告が検出されるたびに、SSAIプラグインはこのレイアウトに切り替わり、広告が終了するとデフォルトのコントローラーに戻ります。

独自のレイアウトを使用する場合、意図しない結果を避けるために、最初に指定したものと同じビュータイプとビューIDを使用します。 Android Studioを使用してAd Media Controllerのレイアウトを調べます。 SSAIプラグインバージョン6.9.0の場合、 default_ssai_ad_media_controller.xml 次のようになります。

<com.brightcove.player.mediacontroller.BrightcoveControlBar
   xmlns:android="http://schemas.android.com/apk/res/android"
   xmlns:tools="http://schemas.android.com/tools"
   android:id="@+id/brightcove_control_bar"
   style="@style/SSAIAdControlBar"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:layout_gravity="bottom"
   android:background="@color/bmc_background"
   android:orientation="vertical"
   android:padding="8dp">

   <LinearLayout
       android:layout_width="match_parent"
       android:layout_height="wrap_content"
       android:gravity="bottom"
       android:orientation="horizontal">

       <Button style="@style/BorderlessButton"
           android:id="@id/play"
           android:layout_width="wrap_content"
           android:layout_height="wrap_content"
           android:text="@string/brightcove_controls_play"
           android:visibility="gone"
           tools:visibility="visible"/>

       <include layout="@layout/ssai_ad_number_countdown_view"/>

       <include layout="@layout/ssai_single_ad_duration_countdown_view"/>

       <View
           android:id="@id/two_line_spacer"
           android:layout_width="0dp"
           android:layout_height="0dp"
           android:layout_weight="1"
           android:visibility="gone"
           tools:visibility="visible"/>

       <Button style="@style/BorderlessButton"
           android:id="@id/full_screen"
           android:layout_width="wrap_content"
           android:layout_height="wrap_content"
           android:padding="4dp"
           android:text="@string/brightcove_controls_enter_full_screen"
           android:visibility="gone"
           tools:visibility="visible"/>
   </LinearLayout>

</com.brightcove.player.mediacontroller.BrightcoveControlBar>

SSAI Ad Media ControllerコンポーネントのビュータイプとビューIDについて説明します。

コンポーネント名 コンポーネントID コンポーネントビュー
再生ボタン play ボタン
全画面ボタン full_screen ボタン
広告番号のカウントダウン text_ad_number_countdown TextView
単一の広告期間カウントダウン text_single_ad_duration_countdown TextView

チェックアウトします カスタマイズされたコントロールのサンプルアプリ、デフォルトのメディアコントローラーがカスタマイズされ、新しいボタンが追加されます。 同じアプローチを適用して、SSAI Ad Media Controllerをカスタマイズできます。

デフォルトのテキストを変更する

すべてのAd Media Controllerコンポーネントは TextViewsアプリで定義されている string.xml ファイル。 表示テキストを変更するには、作成して上書きします string or plurals Native SDK for Androidで使用されるものと同じ文字列IDを使用するアイテム。

挽き目 広告番号のカウントダウン ビューは次の文字列を使用します。

<string name="ssai_message_ad_number_countdown">Ad %1$d of %2$d</string>

挽き目 単一の広告期間カウントダウン ビューは次の文字列を使用します。

<string name="ssai_message_ad_duration_countdown">(%1$s)</string>

サマリー

SSAI UI要素をカスタマイズするいくつかのオプションがあります。 複雑さのレベルは、何を達成したいかによって異なります。 複雑さの低レベルから高レベルまで、次のことができます。

  • VAST設定で特定のコンポーネントを設定/非表示
  • を使用してコンポーネントを表示/非表示 client_options 広告設定のオブジェクト（現在 Live SSAIのみ）
  • 表示される文字列値を変更する
  • UIレイアウトをオーバーライドする

あなたが持っているアイデアやフィードバックを共有してください Brightcove ネイティブプレーヤーSDKフォーラム.
