api_key

api_key:String

APIバージョン： V1、V2

有効な値： きみの Zencoder からのAPIキー https://app.zencoder.com/api

対応するジョブの種類： 全て

例： bcafecadfecabafedac

説明：

あなたのためのAPIキー Zencoder アカウント。 あなたのAPIキーは https://app.zencoder.com/api。 そのページでAPIキーを再生成することもできます。

  {
    "input": "s3://zencodertesting/test.mov",
    "api_key": "bcafecadfecabafedac"
  }

dynamic_profile_options

dynamic_profile_options:Object

APIバージョン： V2

対応するジョブの種類： 全て

説明：

CAEでは、このような出力の数とパラメータが、入力ファイルごとに動的に変更されることがあります。 ザ dynamic_profile_options レンディションを作成する方法を定義します。 見る CAEの設定 詳細については。 また、 generate_dynamic_profile に等しい true この機能を使用する。

  {
    "input": "s3://zencodertesting/test.mov",
    "generate_dynamic_profile": true,
    "dynamic_profile_options": {
      "min_renditions": 2,
      "max_renditions": 8
    },
    "outputs" :[
       {"dynamic_profile_rendition": 1},
       {"dynamic_profile_rendition": 2},
       {"dynamic_profile_rendition": 3},
       {"dynamic_profile_rendition": 4},
       {"dynamic_profile_rendition": 5},
       {"dynamic_profile_rendition": 6},
       {"dynamic_profile_rendition": 7},
       {"dynamic_profile_rendition": 8}
    ]
  }

generate_dynamic_profile

generate_dynamic_profile:Boolean

APIバージョン： V2

有効な値： true または false

デフォルト値： false

対応するジョブの種類： 全て

説明：

CAEでは、このような出力の数とパラメータが、入力ファイルごとに動的に変更されることがあります。 生成される出力数やその他のプロファイル制約の制限を指定するには、CAEジョブ要求にも含める必要があります dynamic_profile_options オブジェクト。 CAE使用のために予約された出力には、 resolution, bitrate、またはコーデック関連のパラメータ。 代わりに、ラベルを含める必要があります dynamic_profile_rendition。 CAE使用のために予約された出力の数は、 max_renditions in dynamic_profile_options.

  {
    "input": "s3://zencodertesting/test.mov",
    "generate_dynamic_profile": true,
    "dynamic_profile_options": {
      "min_renditions": 2,
      "max_renditions": 8
    },
    "outputs" :[
       {"dynamic_profile_rendition": 1},
       {"dynamic_profile_rendition": 2},
       {"dynamic_profile_rendition": 3},
       {"dynamic_profile_rendition": 4},
       {"dynamic_profile_rendition": 5},
       {"dynamic_profile_rendition": 6},
       {"dynamic_profile_rendition": 7},
       {"dynamic_profile_rendition": 8}
    ]
  }

input

input:String

APIバージョン： V1、V2

有効な値： 認証の有無にかかわらず、メディアファイル（HTTP / HTTPS、FTP / FTPS、SFTP、Azure、GCS、CF、またはS3）への有効なURL

対応するジョブの種類： VOD

例：

  • http://example.com/path/to/input.avi
  • sftp：// user：pass@example.com/path/to/input.mp3
  • s3：//my-bucket/video.mp4
  • cf：// username：api_key@container/video.wmv
  • cf + uk：// username：api_key@container/video.wmv
  • azure：//アカウント名：account-key@container/video.wmv
  • gcs：// access_key：secret_key@gcs-bucket/video.mov
  • aspera：// user：pass @ host：ssh-port / path / to / file.mp4
  • s3 + eu-central-1：//my-bucket/video.mp4

説明：

トランスコードするファイルのURL。 リモートサーバーで認証が必要な場合は、入力URL文字列にユーザー名とパスワードを含める必要があります。

URLに特殊文字を含む認証が含まれている場合、 それらが適切にエスケープされていることを確認してください.

S3に関する注意：

絶対です 助成金 書きます への権限 Zencoder 出力バケット上 にジョブを送信する前に Zencoder。 書き込み許可が必要なのはS3バケットであり、S3「パス」ではないことに注意してください。 たとえば、 http://s3.amazonaws.com/my_files/0000/video.mp4 「my_files / 0000」（パス）ではなく、「my_files」（バケット名）に追加された書き込みアクセスが必要です。

URL（？または＃）で有効でない特殊文字を含むキーを作成するには、それらをURLエンコードする必要があります（それぞれ％3Fおよび％23）。 他のすべての文字はURLエンコードされていてはなりません。

S3 URLが事前署名されている（「署名」、「期限切れ」および「AWSAccessKeyId」クエリパラメータを含む）場合、すべての文字をURLエンコードする必要があります。

URLのプロトコルを次のように設定して、S3バケットの領域を指定することができます。 s3+region-nameここで、「地域名」は この文書。 たとえば、アジア太平洋（シンガポール）地域のバケットからファイルにアクセスするには、URLを次のように設定します。 s3+ap-southeast-1://my-bucket/file.mp4.

Azureに関する注意事項：

URLに特殊文字を含む認証が含まれている場合、 それらが適切にエスケープされていることを確認してください.

クラウドファイルに関する注意：

使用するリージョン（DFW、ORD、UK）は、プロトコルに追加することで指定できます。 cf+ord://username:api_key@container/object。 現在サポートされている唯一のクラウドファイル領域はDFW、ORD、UKです。 デフォルトでDFW領域が使用されます。

コンテナ名の中の特殊文字は、URLエンコードされている必要があります。

Google Cloud Storageに関する注意事項：

GCSアクセスキーは、の「相互運用アクセス」セクションにあります。 Google APIコンソール。 アクセスキーは 適切に脱出した.

ドットを含むGCSバケット名はサポートされていません。

Asperaに関する注意：

URLのssh-port部分は必須ではなく、デフォルトで22になります。

Zencoder に接続するビデオエンコーディングサーバーでAsperaクライアントを実行します。 Aspera EnterpriseまたはConnect Serverこれはファイルをホストします。

これは、ファイルを利用できるようにするためにFTPサーバーを実行している場合と同様の状況です。 Zencoder ファイル転送にFTPクライアントを使用します。

  {
    "input": "http://example.com/path/to/input.avi"
  }

outputs

outputs:Array or Hash

APIバージョン： V1、V2

有効な値： ハッシュの配列または単一のハッシュ

対応するジョブの種類： 全て

例：

  • "outputs"：[{}]
  • "output"：{}

説明：

私たちのデフォルト出力は、以下の設定を持つ単一のビデオファイルです：

  • H.264ビデオ、AACオーディオ、MP4ファイル形式
  • 動画： 中品質
  • オーディオ：ステレオ、 中品質 （約112kbps）、44100 Hz

出力場所を指定しないと、ファイルは一時的にホストされます。 Zencoder。 24時間後、ファイルは使用できなくなります。

  {
    "input": "s3://zencodertesting/test.mov",
    "outputs": [
      {
        "url": "s3://my-output-bucket/final-1.mp4"
      },
      {
        "url": "s3://my-output-bucket/final-2.webm"
      }
    ]
  }

region

region:String

APIバージョン： V1、V2

デフォルト： us

有効な値： オレゴン、us-n-california、ユーダブリン、asia-singapore、asia-tokyo、sa-saopaulo、australia-sydney、us-central-gce、 eu-west-gce、またはasia-east-gce

対応するジョブの種類： 全て

例： 私たち - バージニア

説明：

ジョブをエンコードするために使用するAmazon AWS領域を指定することができ、指定された領域のサーバー上でジョブを処理します。

大陸全体の地域を指定できます：us、europe、asia、sa、australia。 これらのいずれかを使用すると、大陸内の任意の地域を使用してジョブを処理できます。

特定のAWS地域をターゲットにするために、より特定の地域を使用することができます：us-virginia、us-oregon、us-n-california、e-dublin、asia-singapore、asia-tokyo、sa-saopaulo、australia-sydney。

  {
    "input": "s3://zencodertesting/test.mov",
    "region": "us-virginia"
  }

test

test:Boolean

APIバージョン： V1、V2

デフォルト： false

有効な値： true または false

対応するジョブの種類： 全て

ジョブのテストモードを有効にする（「統合モード」）。 統合モードでは、すべてのエンコードされたファイルは5秒に短縮され、ジョブに対しては料金はかかりません。

  }

download_connections

download_connections:Integer

APIバージョン： V1、V2

デフォルト： 5

有効な値： 0と25の間の整数

対応するジョブの種類： VOD

説明：

ダウンロードに使用する接続数を指定できます。 input ファイル。 これにより、リモートサーバーの帯域幅に応じて、ダウンロード転送時間が短縮される場合があります。 より多くの接続がサーバーに重い負荷をかける可能性があることに注意してください。 ダウンロードタイムアウトに問題がある場合、または防止したい場合 Zencoder ファイルを取得するときにあまりにも多くの帯域幅を使用することから、これを1に設定します。

  {
    "input": "s3://zencodertesting/test.mov",
    "download_connections": 3
  }

grouping

grouping:String

APIバージョン： V1、V2

有効な値： 任意の文字列。

対応するジョブの種類： 全て

例： 私のカスタムパススルーバリュー

説明：

任意の文字列をここに渡して、レポートの目的でグループ化することができます。 これにより、各グループの統計情報が収集されます。 レポートAPI.

aspera_transfer_policy

aspera_transfer_policy:String

APIバージョン： V1、V2

デフォルト： フェア

有効な値： 公正または低い

対応するジョブの種類： VOD

説明：

方法を設定する Zencoder Asperaサーバーからファイルを転送するときに、エンコードサーバーの使用可能な帯域幅を使用します。

フェア エンコードサーバー上のすべての実行中のジョブ間で帯域幅を均等に分割します。

ロー エンコーディングサーバーで使用可能な帯域幅を使用しますが、他のジョブに優先順位を与えます。 低すぎると、サーバーのオーバーランが懸念される場合に役立ちます。

注：AsperaのHighオプションは、 Zencoder。 デフォルトではできるだけ多くの帯域幅を使用しますが、エンコーディングサーバー上の他のすべてのものに対して1つのジョブの帯域幅へのアクセスに優先順位をつけません。

  {
    "input": "s3://zencodertesting/test.mov",
    "aspera_transfer_policy": "fair"
  }

transfer_minimum_rate

transfer_minimum_rate:Number

APIバージョン： V1、V2

デフォルト： 1000

有効な値： 1000-249999

対応するジョブの種類： VOD

説明：

Asperaでのファイル転送中に使用する転送レートフロアを設定します。 これは速度に達することを保証するものではありませんが、帯域幅が利用可能な場合、ファイル転送はこの速度を下回らないでしょう。

これは、 transfer_maximum_rate.

  {
    "input": "s3://zencodertesting/test.mov",
    "transfer_minimum_rate": 2500
  }

関連項目： transfer_maximum_rate や aspera_transfer_policy

transfer_maximum_rate

transfer_maximum_rate:String

APIバージョン： V1、V2

デフォルト： 250000

有効な値： 1001-250000

対応するジョブの種類： VOD

説明：

Asperaでのファイル転送中に使用する転送レートの上限を設定します。 これは速度に達することを保証するものではありませんが、帯域幅が利用可能な場合、ファイル転送はこの速度を超えません。

大量のジョブを同時にエンコードする場合は、送信元サーバーと宛先サーバーのトラフィックが圧倒的に大きくならないように、最大​​転送レートを設定することをお勧めします。

これはより大きい必要があります transfer_minimum_rate.

  {
    "input": "s3://zencodertesting/test.mov",
    "transfer_maximum_rate": 25000
  }

expected_md5_checksum

expected_md5_checksum:String

APIバージョン： V2

対応するジョブの種類： VOD
