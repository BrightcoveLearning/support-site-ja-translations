このトピックでは、Android Studioと Brightcove Player Android用のSDKを使用して、 Brightcove Video Cloud アカウント、および Brightcove Video Cloud プレイリスト。

このトピックでは、Android Studioと Brightcove Player Android用SDKを使用して、1つの動画を再生するアプリを作成します。

概要

このクイックスタートを完​​了すると、次のことができるようになります。

  • プロジェクトを作成し、 Brightcove Player Gradleを使用したAndroid用SDK
  • レイアウトを変更して BrightcoveExoPlayerVideoView.
  • に追加する onCreate() 動画を再生する方法. 異なるソースから。
ノート：
  • この例では、 BrightcoveExoPlayerVideoViewこれは、ほとんどの場合に使用するのに推奨されるビューです。 別のビューを使用する必要があると思われる場合は、 BaseVideoViewの選択 の資料をご参照ください。
  • この例では、 Brightcove Playback API。 これは、最新の推奨APIで、 Video Cloud としょうかん。

Audience

開発用にAndroid Studioを使用していて、 Brightcove Player AndroidアプリのAndroid用SDK

必須条件

JavaとAndroidアプリの開発についての知識はほとんどありません。

準備をしなさい

Android Studioをインストールして開発の準備をする

  1. ダウンロード Androidのメーカー アプリケーション。
  2. ページの指示に従って、アプリケーションをインストールします。

プロジェクトを作成する

Android Studioでプロジェクトを作成し、Gradleを使用してプレーヤーSDKにリンクします。

Android Studioでプロジェクトを作成する

  1. Android Studioを開きます。
    • Android Studioを初めて開く場合は、 新しいAndroid Studioプロジェクトを開始する.
    • メニューが表示されている場合は、 ファイル>新規>新規プロジェクト.
  2. の中に プロジェクトを選択してください ウィンドウ、選択 空の活動 をクリックし 次.

  3. あなたの 氏名, パッケージ名, Languageおよび 位置を保存。 このクイックスタートでは、以下の値が使用されます。

  4. ナビゲーションヘッダーの 仕上げ.

  5. Android Studioはしばらくの間動作し、最終的にプロジェクトの初期状態を表示します。

Gradleを利用して Brightcove Player Android用SDK

GradleはAndroid Studioと統合されているため、Android用Native SDKをプロジェクトに追加することができます。

  1. 拡大する Gradle Scripts セクションを開き、2番目のセクションを開きます build.gradle このファイルは、PlayVideosプロジェクトの現在のモジュールに関連付けられています。

  2. の中に build.gradle ファイルを作成するには、次の操作を行います。

    1. の中に android セクション、次を追加 compileOptions 値：
      ExoPlayer v2.9.0 +の場合、Java 8サポートを有効にする必要があります。 これを行うには、以下を追加します android アプリのセクション build.gradle ファイル。 詳細については、 AndroidのExoPlayer の資料をご参照ください。
      android {
    compileSdkVersion 28
    defaultConfig {
        applicationId "com.brightcove.playvideos1"
        minSdkVersion 21
        targetSdkVersion 28
        versionCode 1
        versionName "1.0"
        testInstrumentationRunner "androidx.test.runner.AndroidJUnitRunner"
    }
    buildTypes {
        release {
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
    compileOptions {
        targetCompatibility JavaVersion.VERSION_1_8
    }
}
    2. 以下 android セクション、追加 repositories のセクション Brightcove Mavenレポ。 
      repositories {
      maven {
          url 'https://repo.brightcove.com/releases'
      }
    }
    3. の中に dependencies Android用Native SDKの最新バージョンを追加してください。 値を置き換える 6.10.0 下記の最新のSDKバージョンをご覧ください。 概要： Brightcove Android 用ネイティブ SDK の資料をご参照ください。 
      dependencies {
  implementation fileTree(dir: 'libs', include: ['*.jar'])
  implementation 'com.android.support:appcompat-v7:26.1.0'
  implementation 'com.android.support.constraint:constraint-layout:1.0.2'
  testImplementation 'junit:junit:4.12'
  androidTestImplementation 'com.android.support.test:runner:1.0.1'
  androidTestImplementation 'com.android.support.test.espresso:espresso-core:3.0.1'
  implementation "com.brightcove.player:exoplayer2:6.10.0"
}

    Gradleコミュニティでは、「+」文字を使用した動的依存関係バージョンの使用は推奨されていません。 動的バージョン管理を使用すると、新しいAPIがアプリのソースコードと予期せず互換性がなくなる可能性があるため、ビルドプロセスに重大なリスクをもたらします。

    ノート：
    • Brightcove 特定のバージョン文字列で常に最新バージョンを使用することをお勧めします。 たとえば、 build.gradle や gradle.properties のファイル アンドロイドプレイヤーサンプル プロジェクト。
    • ExoPlayer依存関係にはAndroidのネイティブSDKが含まれているため、SDKを個別に含める必要はありません。
    • GoogleのExoPlayer Android用のオープンソースのアプリケーションレベルのメディアプレーヤーです。 ExoPlayer 2フレームワークを使用していることに注目してください。 詳細については、 ExoPlayer 2フレームワークへの移行 の資料をご参照ください。
  3. プロジェクトを同期させて、Gradleファイルの変更をピックアップします。

  2. ナビゲーションヘッダーの テキスト ファイルのタブの下部にある、生のXMLを確認します。

  3. 既存のものを削除する TextView 要素を追加し、 BrightcoveExoPlayerVideoView 結果のXMLは次のように表示されます。 後で使用するための注意 id ビューの名前は brightcove_video_view.

アプリでインターネットを使用できるようにする

  1. Video Cloud Studioで app/manifests/AndroidManifest.xml ファイル。

  2. ちょうど <application> コードブロックですが、終わりより上です </manifest> インターネット接続を有効にするには、次のように入力します。

ビデオの取得と再生

このセクションでは、 Catalog クラスから単一のビデオを取得する Video Cloud サーバー、次にそれを再生します。

挽き目 com.brightcove.player.edge.Catalog クラスは、ビデオやプレイリストを非同期メソッドから取得します。 Brightcove Playback API。 これは、最新の推奨APIで、 Video Cloud としょうかん。

カタログからビデオを取得する

  1. XNUMXサーバ当たり年間 Video Cloud スタジオで、次の情報を収集します。
    • アカウントID
    • 動画ID
    • ポリシーキー
    ノート：
    • アクセスするために Playback APIポリシーキーが必要になります。 ポリシーキーに精通していない場合は、 Policy API 概要.
    • ネイティブSDKはオリジンヘッダーを送信しないため、ドメイン制限されたポリシーキーを使用することはできません。 これを試してみると、 403 Forbidden コンテンツを取得しようとすると Video Cloud としょうかん。

  2. プロジェクトでカスタム値を定義します。 オープン res/values/strings.xml ファイルを更新し、次のように更新します。

      <!-- A sample Brightcove Edge Policy Key -->

      <!-- A sample Brightcove Video ID -->

  3. に戻る MainActivity.java ファイルを開き、次のコードを追加します。

    • SDKからイベントエミッタを取得する

    • のインスタンスを作成する com.brightcove.player.edge.Catalog前の手順のアカウントIDとポリシーキーに定義された値を使用します。

プレイリストを取得して再生する

このセクションでは、 Catalog クラスからプレイリストを取得する Video Cloud サーバーを開き、プレイリストでビデオを再生します。

挽き目 com.brightcove.player.edge.Catalog クラスは、ビデオやプレイリストを非同期メソッドから取得します。 Brightcove Playback API。 これは、最新の推奨APIで、 Video Cloud としょうかん。

不要なコードを削除する

  1. 以前のアプリからのコードの一部が不要になりました。 カタログの呼び出しを削除する findVideoByID() メソッド、および関連する VideoListener 匿名コールバック関数。

    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

    // Add the video found to the queue with add().
    // Start playback of the video with start().
    @Override
      public void onVideo(Video video) {
        brightcoveVideoView.add(video);
        brightcoveVideoView.start();
      }

    @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
  });

カタログからプレイリストを取得する

  1. 既存のカタログインスタンスはプレイリストを取得するために機能します。 変更は必要ありません 次のコード行に追加します。

  2. In Video Cloud スタジオ 언론방송 モジュールを選択し、プレイリストを選択して プレイリストID.

  3. カタログの使用 findPlaylistByID() あなたのプレイリストIDと PlaylistListener コールバックのために。

  }

  }

    挽き目 PlaylistListener 非同期操作からプレイリストオブジェクトを受け取ることを期待するオブジェクトを識別します。 成功した操作では、 onPlaylist() メソッドが呼び出されます。 操作にエラーがある場合は、 onError() メソッドが呼び出されます。 Android Studioは、必要な2つのメソッドのシグネチャを挿入しました。

  4. の中に onPlaylist() メソッドを使用して、プレイリストからビデオを取得し、すべてのビデオを brightcoveVideoView最初の動画を開始します。
